# M2 — 基础策略 (Basic Strategies)

> 状态:**⬜ 待实现**

## 目标
在地基之上实现 T1~T3 人类策略,使引擎能不依赖暴力解开 easy~hard 绝大多数题,
并用全量语料统计解出率,据此确认/调优策略顺序(即"系统化流程"本体)。

对应总纲:FR-4(T1-T3)、FR-5、FR-7、AC-2、AC-3、AC-4。

## 范围内策略
- **T1 单数(difficulty 10)**:full house、naked single(参考已有)、hidden single
- **T2 区块与数组(difficulty 20~30)**:
  - locked candidates:pointing(宫→行/列)、claiming(行/列→宫)
  - naked subsets:pair / triple / quad
  - hidden subsets:pair / triple / quad
- **T3 鱼/翼/单数字模式(difficulty 40~50)**:
  - 基础鱼:X-Wing / Swordfish / Jellyfish(统一用 base/cover 集合模型)
  - 单数字模式:Skyscraper / 2-String Kite / Empty Rectangle
  - 翼:XY-Wing(Y-Wing)/ XYZ-Wing / W-Wing

规则来源:`research/sudoku-human-solving/local-library/techniques/`。
术语来源:`research/sudoku-human-solving/glossary.zh-en.md`。

## 实现约定
- 每个策略 = 一个独立纯函数模块,导出符合 `Strategy` 接口的对象(模仿 `naked-single.ts`)。
- `apply` 返回首个可行步或 null,**不得修改 grid**;必须填好 `highlights` 与双语 `explanation`。
- 在 `strategies/index.ts` 的 `STRATEGIES` 数组按 difficulty 顺序注册。

## 验收清单
- [ ] 每个策略有单元测试,用手工最小盘面断言精确产出(**AC-2**)
- [ ] **AC-3 健全性回归**:对 `data/ground-truth/` 全部 400 题,引擎走的每一步都通过
      `checkTraceSoundness`(零 violation)
- [ ] **AC-4 解出率**:跑全量(或大样本)语料,按难度报告非暴力解出率;
      easy 期望接近 100%,medium/hard 显著提升
- [ ] 新增"解出率统计脚本"(`scripts/solve-rate.ts`,产出 `data/reports/solve-rate.json`)
- [ ] typecheck + 全部单测通过
- [ ] 策略顺序经统计调优后写入 `STRATEGIES`(便宜优先)
