# M1 — 地基 (Foundation)

> 状态:**✅ 已完成**(由基线在 `foundation` 分支建立,作为所有模型分支的统一起点)

## 目标
搭好所有后续工作共享的不可变地基:盘面模型、谜题解析、暴力校验/标准答案、
SolveTrace 数据契约、策略接口、求解循环、健全性校验,以及测试与冻结的评分数据集。

对应总纲:`docs/requirements.md` 的 FR-1~3、FR-5、FR-6、NFR-1、AC-1。

## 详细需求
- **盘面模型** `packages/engine/src/grid.ts`:81 格 row-major;候选用 9-bit mask;
  预计算 `HOUSES`(27)、`PEERS_OF`(每格 20)、`UNITS_OF`、行/列/宫索引;
  `place` 落子时清理 peer 候选(仅规则记账,不做推理)。
- **谜题解析** `parser.ts`:正则解析 OpenSudoku XML 的 `<game data="…81…">`,带 folder 归属。
- **暴力校验** `bruteforce.ts`:MRV + 位运算回溯;`solveBruteforce`、`countSolutions`、
  `findGroundTruth`(解 + 唯一性)。**仅用于验证,绝不作教学步骤。**
- **数据契约** `trace.ts`:`CellDigit / Link / Highlights / Step / SolveTrace`。**冻结接口。**
- **策略接口** `strategy.ts`:`Strategy { id, name{zh,en}, difficulty, apply(grid): Step|null }`。
  这是多模型对比的接缝(seam)。
- **求解循环** `solver.ts`:按 difficulty 升序试策略,取首个产出步并应用,记录为 Step,
  循环至解出/卡死;`applyStep` 应用 placements/eliminations。
- **健全性校验** `soundness.ts`:`checkTraceSoundness(trace, solution)` —— 任何与真解矛盾的
  落子/消除都报为 violation。**项目最强自动化护栏,也是模型横评主信号。**
- **参考策略** `strategies/naked-single.ts`:唯一随地基交付的策略,固定 Step 构造约定供模型分支模仿。
- **测试** `test/*.test.ts`:grid / parser / bruteforce / solver+soundness 单测 + 真实语料集成测试。
- **标准答案生成** `scripts/gen-ground-truth.ts`:确定性抽样(固定 stride/数量,无随机),
  每档 100 题,产出 `data/ground-truth/<difficulty>.json`,作冻结评分集。
