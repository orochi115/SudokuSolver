# M5 — 交互式引导 UI (Interactive Tutor)

> 状态:**⬜ 待实现**

## 目标
从任意盘面出发,提供辅助功能引导用户用各种策略解题:分级提示、落子校验、错误讲解。

对应总纲:FR-13~16、AC-6。

## 详细需求
- **任意盘面输入**(FR-13):支持用户手填/部分填写;由 `Grid` 重算候选。
- **渐进式提示**(FR-14):对当前盘面调引擎求"下一可行 Step",分三级披露:
  1. 技巧类别(如"试试区块排除")
  2. 关注区域(高亮相关 house/cells,不给答案)
  3. 确切一步(给出 placement/elimination 与讲解)
- **落子校验**(FR-15):用户落子后判定对错;错误时解释原因(可借 `checkTraceSoundness` 的思路,
  或对照引擎当前可行步/标准答案)。
- **复用**(FR-16):棋盘、高亮、讲解面板复用 M4 组件;提示数据仍是 `Step` 结构。

## 验收清单
- [ ] 能从任意(含用户自定义)盘面计算并给出分级提示
- [ ] 三级提示逐级披露,且最终一步与引擎一致
- [ ] 落子校验正确,错误有可理解的原因
- [ ] **AC-6**:可用浏览器 MCP 走查 —— 任意盘面给提示 + 故意落错子触发纠错
- [ ] 复用 M4 组件,无重复解题逻辑

## 依赖
- M4(回放组件)、M1~M3(引擎与策略,提示质量取决于策略覆盖广度)。

## 备注
- "下一可行 Step"直接复用 `solver` 的单步:对当前 grid 按 difficulty 升序试策略,返回首个产出。
  这意味着提示天然遵循"系统化流程"的顺序。
