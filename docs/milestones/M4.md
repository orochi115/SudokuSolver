# M4 — 网页逐步回放 (Step Replay)

> 状态:**⬜ 待实现**

## 目标
做一个网页:加载谜题 → 引擎产出 SolveTrace → 候选数网格渲染 → 逐步回放完整解题过程,
高亮每步的格/候选/链路径,并显示双语讲解。

对应总纲:FR-10~12、AC-5。

## 详细需求
- 新建 `packages/web`(Vite + TS),直接 `import { solve, Grid, STRATEGIES } from '@sudoku/engine'`,**无后端**。
- **棋盘组件**:9×9 网格,显示已填数字与候选数小标记。
- **回放控制**:上一步 / 下一步 / 跳转;从 `SolveTrace.initial` 累积重建每步状态(增量,不存全量快照)。
- **高亮层**:渲染当前 `Step.highlights` 的 cells、candidates、links(链画成连线,强/弱不同样式)。
- **讲解面板**:显示当前 `Step.explanation`(中/英可切换),术语对齐 glossary。
- **谜题来源**:可粘贴 81 字符串,或从语料/标准答案集选题。

## 验收清单
- [ ] `packages/web` 脚手架(Vite)可 `npm run dev` 启动
- [ ] 能加载任一谜题并展示引擎 SolveTrace 的逐步回放
- [ ] **AC-5**:高亮(含链路径)与双语讲解正确对应每一步
- [ ] 可用 chrome-devtools / playwright MCP 走查若干题确认渲染
- [ ] 组件结构为 M5 复用预留(棋盘、高亮、讲解面板解耦)

## 依赖
- M1(SolveTrace)。可在 M2/M3 进行中并行启动(先用已实现的策略集回放)。

## 备注
- 这一层应是"哑"的:只消费 SolveTrace,不含任何解题逻辑,从而与引擎实现完全解耦,
  也使多模型分支的引擎都能用同一套回放 UI。
