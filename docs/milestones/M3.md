# M3 — 高级策略 (Advanced Strategies)

> 状态:**⬜ 待实现**(项目 ~80% 工程量集中于此)

## 目标
实现 T4 高级技巧,使引擎能解开 diabolical 档;核心是把盘面建模为**强/弱链接图**并搜索 AIC,
named 技巧多作为其特例。同时成文 forcing-chain 边界规则,并派生人读流程文档。

对应总纲:FR-4(T4)、FR-8、FR-9、AC-3、AC-4。

## 范围内策略(建议实现顺序)
1. **简单染色 Simple Coloring**(difficulty 60):单数字强链的双色,trap/wrap。
2. **AIC 引擎 + 链类**(difficulty 70):
   - 通用强/弱链接图底座(候选为节点,强链/弱链为边,支持 grouped links)
   - X-Chain、XY-Chain、Nice Loops、AIC(Type 1/2、连续/不连续环)
   - named 特例:Skyscraper/Kite/Turbot/Empty Rectangle 可由短链统一解释
3. **ALS(difficulty 80)**:ALS-XZ、doubly-linked ALS、ALS-XY-Wing、ALS 链、Death Blossom。
4. **唯一性(difficulty 90,标记 optional)**:Unique Rectangle、Avoidable Rectangle、BUG/BUG+1。
   *依赖"唯一解假设",在引擎中以开关控制是否启用。*
5. **特殊(difficulty 95)**:Sue de Coq、aligned exclusion(按需)。
6. **forcing chains/nets(difficulty 100,最后手段)**:受 FR-8 边界规则约束。

## FR-8 forcing-chain 边界规则(本里程碑必须先定稿)
在实现链类的"分叉/枚举"形态前,产出 `docs/forcing-boundary.md`,明确:
- 何种链算"人类可接受逻辑"(如:单一前提沿强弱交替推进、不分叉、深度 ≤ 阈值)
- 何种算"伪装的枚举"并应禁用或显式标记(如:多分叉 forcing nets、Nishio 试错、模板枚举)
- 引擎以配置项暴露该边界,使"是否允许某类链"可统一控制。
