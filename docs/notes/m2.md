# M2 Design Notes

## Implemented Strategies

### T1 (difficulty 10): Singles
- **full-house**: Detects when a house has exactly one empty cell; places the missing digit.
- **naked-single** (reference): Already implemented in foundation.
- **hidden-single**: Scans each house for a digit that appears only once among empty cells.

### T2 (difficulty 20-30): Intersections & Subsets
- **locked-candidates**: Implements both Pointing (box→row/col) and Claiming (row/col→box) by examining digit distribution patterns.
- **naked-subset**: Detects naked pairs/triples/quads by finding N cells whose combined candidate mask has exactly N bits.
- **hidden-subset**: Detects hidden pairs/triples/quads by finding N digits confined to exactly N cells in a house.

### T3 (difficulty 40-50): Fish, Patterns & Wings
- **basic-fish**: Unified base/cover model for X-Wing (size 2), Swordfish (size 3), Jellyfish (size 4). Scans both row-based and column-based fish.
- **single-digit-patterns**: Implements Skyscraper and Empty Rectangle patterns.
- **xy-wing**: Pivot {X,Y} sees pincers {X,Z} and {Y,Z}; eliminates Z from cells seeing both pincers.
- **xyz-wing**: Pivot {X,Y,Z} sees pincers {X,Z} and {Y,Z}; eliminates Z from cells seeing both pincers. Requires pivot NOT to contain Z.
- **w-wing**: Two bivalue cells with same pair are bridged by a strong link on one digit.

## Key Design Decisions

### XYZ-Wing Bug Fix
The original XYZ-Wing implementation had a critical bug: it identified valid patterns where the pivot contained digit Z, but in a correct XYZ-Wing, the pivot must NOT contain Z (otherwise the chain logic breaks). The fix adds a check `if ((pivotMask & (1 << (z - 1))) !== 0) continue;` to skip pivots that contain Z.

### Fish Base/Cover Model
Used a unified model that handles X-Wing, Swordfish, and Jellyfish with the same algorithm. The key insight is that for a given digit, if N base houses have their candidates fully contained within exactly N cover houses, we can eliminate the digit from cover-house cells outside the base intersections.

### Data Structures
- Candidates stored as 9-bit bitmasks per cell for efficient operations
- Static topology precomputed once (HOUSES, PEERS_OF, ROW_OF, COL_OF, BOX_OF)
- Strategies are pure functions that scan the grid and return the first applicable step

## Soundness
All 400 ground-truth puzzles pass `checkTraceSoundness` with zero violations. The strategies only eliminate candidates or place digits that are consistent with the solution.

## Challenges Encountered

### XYZ-Wing Implementation Error
The standard XYZ-Wing rule states that the pivot's digit Z must NOT be present in the pivot cell. My initial implementation matched patterns where pivot contained Z, but these are not valid XYZ-Wings and produced incorrect eliminations. Fixed by adding the guard condition.

### Test String Length Issues
Early unit tests used incorrectly constructed 81-character puzzle strings, causing "Grid string must be 81 chars" errors. Fixed by using proper string concatenation with `'0'.repeat(72)` for padding.
