# M2 设计说明

## 实现范围

本次在 M1 地基上完成了 M2 要求的 T1~T3 策略模块，并统一遵守 `Strategy` 接口与 `Step` 产出约定（包含 highlights 与中英 explanation）：

- T1
  - `hidden-single`
- T2
  - `locked-candidates`（覆盖 pointing / claiming）
  - `naked-pair` / `naked-triple` / `naked-quad`
  - `hidden-pair` / `hidden-triple` / `hidden-quad`
- T3
  - 基础鱼：`x-wing` / `swordfish` / `jellyfish`
  - 单数字模式：`skyscraper` / `two-string-kite` / `empty-rectangle`
  - wings：`xy-wing` / `xyz-wing` / `w-wing`

并在 `packages/engine/src/strategies/index.ts` 按 difficulty 升序注册到 `STRATEGIES`。

## 核心思路

- `hidden-single`
  - 扫描 27 houses × 9 digits，若某 digit 在某 house 只剩一个候选位置，则直接 placement。

- `locked-candidates`
  - Pointing：box 内某 digit 候选全部落在同一行/列，则删去该行/列宫外同 digit。
  - Claiming：行/列内某 digit 候选全部落在同一 box，则删去该 box 行/列外同 digit。

- naked / hidden subsets
  - 用统一组合枚举：
    - naked subset：枚举 house 内 size 个格子，候选并集位数恰为 size，再从 house 其余格删除并集 digit。
    - hidden subset：枚举 size 个 digit，若它们在 house 的承载格数量恰为 size，则从这些格子删除非该集合候选。

- 基础鱼（X-Wing/Swordfish/Jellyfish）
  - 统一 base/cover 模型：
    - 对某 digit，选择 size 条 base 线（行或列），其候选覆盖到的对向线数量恰为 size。
    - 在 cover 线中、base 线之外执行 eliminations。

- Skyscraper / 2-String Kite / Empty Rectangle
  - 统一为 turbot 模式：两条 strong links + 一条 weak 连接，推出两端尾点形成排除。
  - 三者通过 strong-link 组合类型与 weak 连接类型区分（行列/宫区块组合约束）。

- XY / XYZ / W-Wing
  - `xy-wing`：双值 pivot + 两个双值 pincers（共享 z）进行共同可见格排除。
  - `xyz-wing`：三值 pivot + 两个双值 pincers，排除同时可见 pivot 与两翼的 z。
  - `w-wing`：同对双值单元，经某 digit 的 strong link 桥接，删除另一 digit。

## 关键设计决策与取舍

- 基础鱼采用统一 base/cover
  - X-Wing/Swordfish/Jellyfish 仅在规模上不同，抽象后逻辑一致，减少重复代码并便于后续扩展更复杂鱼形。

- subsets 统一工厂化
  - pair/triple/quad 共享一套核心检测流程，仅通过 size 参数区分，避免三套近似实现漂移。

- 单数字模式统一为 turbot 框架
  - 把 skyscraper / kite / empty rectangle 归并到“strong-weak-strong”链式模板，提高复用与可维护性。
  - 该实现偏保守，优先保证 soundness（不产生与真解冲突的步）。

- 数据结构选择
  - 继续使用 Grid bitmask（候选）与 houses 预计算；策略内部只做纯读扫描与组合枚举。
  - 统一使用 `Step` 的增量输出（placements/eliminations）与 highlights，保持 M4/M5 复用契约。

## 解出率（非暴力）

数据来源：`data/reports/solve-rate.json`（由 `packages/engine/scripts/solve-rate.ts` 生成）。

- easy: 100/100 = 100%
- medium: 100/100 = 100%
- hard: 87/100 = 87%
- diabolical: 14/100 = 14%
- overall: 301/400 = 75.25%

`easy` 达到预期“接近 100%”。

## 难点与处理

- 模式类策略容易出现实现分叉
  - 处理：将 subsets / fish / turbot / wings 以“可复用核心 + 薄策略壳”组织，减少重复与错漏。

- 声明正确但实现不稳的风险（尤其 T3）
  - 处理：新增 ground-truth 全量 soundness 回归测试，对 400 题 trace 全量做 `checkTraceSoundness`，要求零 violation。

- 测试可构造性
  - 处理：每个策略均配最小手工候选盘面测试，断言 strategyId 与精确 eliminations/placements，覆盖 T1~T3 全策略。
