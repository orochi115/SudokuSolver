# M2 设计说明

## 本次实现范围

已在 `packages/engine/src/strategies/` 实现并注册以下策略:

- T1: `hidden-single`（`naked-single` 继续作为参考实现）
- T2: `locked-candidates`（pointing + claiming）、`naked-subset`（pair/triple/quad）、`hidden-subset`（pair/triple/quad）
- T3:
  - 基础鱼: `x-wing`、`swordfish`、`jellyfish`
  - 单数字模式: `skyscraper`、`two-string-kite`、`empty-rectangle`
  - wings: `xy-wing`、`xyz-wing`、`w-wing`

并在 `packages/engine/src/strategies/index.ts` 中按 difficulty 升序注册到 `STRATEGIES`。

## 核心实现思路

### 1) House / 组合扫描框架

新增 `packages/engine/src/strategies/common.ts` 提供复用能力:

- 组合枚举 (`combinations`)
- 单元关系判断 (`sees`, `peersIntersection`)
- 去重 (`uniqueCells`, `uniqueEliminations`)
- house 与候选扫描辅助

这样每个策略保持独立模块、纯函数 `apply(grid) => Step | null`，且不修改 grid。

### 2) T2 子集策略统一实现

- `naked-subset`: 以 house 为单位枚举大小 2/3/4 的单元组合，判断并集候选数是否正好等于组合大小。
- `hidden-subset`: 以 house 为单位枚举大小 2/3/4 的数字组合，判断这些数字是否只落在同样数量的单元格中。

两者都返回首个可行步，并填充 `highlights` / 中英解释。

### 3) 基础鱼统一 base/cover 模型

新增 `packages/engine/src/strategies/fish-common.ts`，用同一套 base/cover 逻辑实现:

- X-Wing(size=2)
- Swordfish(size=3)
- Jellyfish(size=4)

通过参数化 `size` 与名称复用扫描过程，满足 M2 对“统一 base/cover 集合模型”的要求。

### 4) 单数字模式统一为 Turbot-like 链核

新增 `packages/engine/src/strategies/turbot-common.ts`，先抽取“单数字共轭对(强链) + 弱链接”的通用判定，再按链接类型过滤为:

- `skyscraper`: 行-行 或 列-列
- `two-string-kite`: 行-列
- `empty-rectangle`: 宫-行/列

该设计在保持 soundness 的前提下避免了三个策略重复写链推理逻辑。

### 5) Wings

- `xy-wing`: pivot 双值 + 两个双值翼，消去共同翼数字
- `xyz-wing`: pivot 三值 + 两个双值翼，消去可同时看见 pivot 与两翼的共同数字
- `w-wing`: 两端同双值，借一条单数字强链桥接，消去另一数字

## 关键设计决策与取舍

1. **鱼类采用统一 base/cover 内核**
   - 优点: X/Swordfish/Jellyfish 代码统一、行为一致，便于 M3 扩展复杂鱼。
   - 取舍: 当前仅实现基础鱼，不含 finned/sashimi 等变体。

2. **Skyscraper / Kite / ER 共享 Turbot-like 链引擎**
   - 优点: 用“强-弱-强”单数字链统一表达，稳定且易验证 soundness。
   - 取舍: `empty-rectangle` 当前以“宫 + 线”链模板落地，偏向稳健实现而非覆盖所有教材变体写法。

3. **测试采用手工最小候选盘面**
   - 通过 `makeCandidateGrid` 手工构造最小模式，直接断言精确产出，降低与完整题目上下文耦合。

## 难点与处理

1. **T3 模式多、形态近似**
   - 处理: 把单数字模式抽象为统一链核 + 命名过滤，先保证 soundness 和可维护性。

2. **策略步构造一致性（highlights + 双语 explanation）**
   - 处理: 复用公共工具，所有策略都按 `naked-single` 模板返回结构化 Step。

3. **全量 AC-3 回归成本**
   - 处理: 增加 `soundness-regression.test.ts` 对 `data/ground-truth/` 400 题全跑并断言零 violation。
