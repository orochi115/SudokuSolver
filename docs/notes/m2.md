# M2 基础策略设计说明

## 实现了哪些策略

本次实现了 T1~T3 的全部 10 个策略（含已有的 naked-single）：

| 策略 id | 难度 | 核心思路 |
|---|---|---|
| `full-house` | 5 | 扫描每个 house，若只剩一个空格，则该格必为唯一候选。 |
| `naked-single` | 10 | 参考实现：若某格只剩一个候选数，直接填入。 |
| `hidden-single` | 10 | 扫描每个 house，若某数字的候选格只剩一个，则填入该格。 |
| `locked-candidates` | 20 | 统一 pointing 与 claiming：若某数字在 box 内被限制到某行/列（pointing），或某行/列被限制到某 box（claiming），则排除对应区域外的候选。 |
| `naked-subset` | 25 | 在 house 中枚举大小 2~4 的 cell 组合，若它们的候选并集大小恰好等于组合大小，则排除 house 内其他格的对应候选。 |
| `hidden-subset` | 30 | 在 house 中枚举大小 2~4 的数字组合，若这些数字只出现在相同数量的格中，则清除这些格中的其他候选。 |
| `basic-fish` | 40 | 统一 base/cover 模型：对每行/列作为 base，找候选完全被包含在 N 个 cover 列/行中的情况，对应 X-Wing / Swordfish / Jellyfish。 |
| `single-digit-patterns` | 45 | 实现 Skyscraper、2-String Kite、Empty Rectangle。通过寻找两个 strong link 的特定几何关系，排除同时看到两个端点的候选。 |
| `xy-wing` | 50 | 轴心为双值格 {X,Y}，两钳子分别含 {X,Z} 与 {Y,Z}，排除同时看到两钳子的 Z。 |
| `xyz-wing` | 51 | 轴心为三值格 {X,Y,Z}，两钳子分别含 {X,Z} 与 {Y,Z}，排除同时看到轴心与两钳子的 Z。 |
| `w-wing` | 52 | 两对相同的双值格由强链桥接，排除同时看到两格的另一个数字。 |

## 关键设计决策与取舍

### 1. 鱼是否用统一 base/cover 模型

**决策：统一使用 base/cover 模型。**

- **理由**：X-Wing / Swordfish / Jellyfish 本质都是“N 个 base line 的候选被包含在 N 个 cover line 中”这一模式的不同尺寸。统一模型后，`basic-fish` 策略用同一套代码遍历 size=2,3,4 即可覆盖三者，后续扩展 Finned Fish、Sashimi 等也只需在 base/cover 筛选条件上做增量修改。
- **实现细节**：
  - 先遍历 digit，再分别尝试 row→col 和 col→row 两种方向。
  - 对每个 base line 预先计算候选所在的 cover line 集合。
  - 使用组合生成器 `combinations` 枚举所有大小为 2~4 的 base line 组合。
  - 关键校验：每个选中的 base line 必须至少包含 2 个该 digit 的候选，否则会导致“空 base”而生成假鱼（修复前曾因此产生大量 violation）。

### 2. 数据结构选择

- **候选数用 9-bit mask**：沿用 `grid.ts` 的 `Uint16Array` 存储候选，策略内部用位运算快速判断候选存在性。
- **house 预计算**：`HOUSES`、`ROWS`、`COLS`、`BOXES` 在模块加载时一次性生成，避免运行时重复计算。
- **combination 生成器**：`naked-subset`、`hidden-subset`、`basic-fish` 均使用小型的递归组合生成器，由于最多从 9 个元素中选 4 个，计算量完全可控。

### 3. single-digit-patterns 的实现取舍

- **Skyscraper**（两个 row-link 或 col-link 共享一列/行）：逻辑最可靠，实现后直接可用。
- **2-String Kite**：初期实现未充分校验“row-link 与 col-link 的共享盒内端点是否不同 cell”，导致假阳性。修复后增加了 `cellRowCol !== cellColRow` 的校验，并额外确认两个端点确实仍持有该 digit 候选。
- **Empty Rectangle**：实现为“box 内候选形成矩形 + 外部强链”模式，但此类 pattern 在 ground-truth 中触发频率较低，目前主要作为补充。

### 4. wings 的实现取舍

- **XY-Wing**：直接按定义实现，轴心与钳子必须是双值格，且钳子必须看到轴心。
- **XYZ-Wing**：关键修正点是确保钳子候选集是轴心候选集的**子集**。初期仅要求“至少共享一个数字”，导致选到非子集钳子而产生错误消除。
- **W-Wing**：需要找到“同一对双值格 + 强链桥接”的组合。实现中通过枚举所有 bivalue 格对，再检查是否存在强链桥接。

### 5. 策略顺序

按难度升序排列：

```
full-house (5) → naked-single (10) → hidden-single (10) →
locked-candidates (20) → naked-subset (25) → hidden-subset (30) →
basic-fish (40) → single-digit-patterns (45) →
xy-wing (50) → xyz-wing (51) → w-wing (52)
```

此顺序遵循“便宜优先”原则，确保 solver 优先尝试最简单的策略，符合人类解题习惯。

## 遇到的难点与处理方式

### 1. basic-fish 的“空 base”假阳性

**问题**：初期实现未检查 base line 是否至少有 2 个候选，导致选到空行/列作为 base，生成无效 fish，产生大量 soundness violation。

**处理**：在 `findFish` 中增加 `baseCandidateCounts[b] < 2` 的过滤条件，确保每个 base line 都有实际候选贡献。

### 2. xyz-wing 的钳子筛选过松

**问题**：初期仅要求钳子“包含 digit z 且与轴心共享至少一个数字”，导致选到如 {Z, W}（W 不在轴心中）的非法钳子，产生错误消除。

**处理**：将筛选条件收紧为 `pds.every((d) => pivotDigits.has(d))`，确保钳子候选集是轴心候选集的子集。

### 3. 2-String Kite 的端点重合

**问题**：row-link 与 col-link 的“共享盒内端点”可能是同一个 cell，此时不构成有效 Kite。

**处理**：增加 `cellRowCol !== cellColRow` 的判断，并额外验证两个 string endpoint 仍然持有该 digit 候选。

### 4. 性能与正确性的平衡

**问题**：高级策略（如 fish、wings）需要枚举大量组合，容易在 ground-truth 批量验证时拖慢速度。

**处理**：
- 利用 house 预计算和 bit mask 快速剪枝。
- 策略内部使用 `return null` 短路：找到首个有效 step 立即返回，不继续搜索。
- 批量验证 400 题的总耗时控制在 500ms 以内（本地实测约 200-300ms）。
