# M2 设计说明 — 基础策略

## 实现的策略

| ID | 名称 | difficulty | 核心思路 |
|---|---|---|---|
| `full-house` | 全屋唯一 | 10 | 一个 house 中只剩一个空格时，其值必然是 1-9 中缺失的数字 |
| `naked-single` | 显性唯一 | 10 | 一个格子的候选数只剩一个，直接填入（参考实现） |
| `hidden-single` | 隐性唯一 | 10 | 一个数字在某个 house 中只有一个格子可以放 |
| `locked-candidates` | 区块排除 | 20 | Pointing：宫内的候选全部在一行/列中，排除该行/列的其他格；Claiming：行/列的候选全部在一个宫中，排除该宫的其他格 |
| `naked-subset` | 显性数组 | 25 | N 个格子有 N 个候选总和，从该 house 其他格子排除这些候选（pair/triple/quad） |
| `hidden-subset` | 隐性数组 | 30 | N 个数字在该 house 只出现在 N 个格子中，排除这 N 个格子的其他候选（pair/triple/quad） |
| `basic-fish` | 基础鱼 | 40 | 统一 base/cover 模型：N 个 base house 的候选被 N 个 cover house 完全覆盖，排除 cover house 中非 base 交集的候选（X-Wing/Swordfish/Jellyfish = size 2/3/4） |
| `single-digit-patterns` | 单数字模式 | 45 | 摩天楼(Skyscraper)：两个强链端点的共同影响格排除该数字（暂未实现 2-String Kite 和 Empty Rectangle） |
| `xy-wing` | XY翼 | 50 | 枢纽 {X,Y} 连接两个钳子 {X,Z} 和 {Y,Z}，排除共同影响格中的 Z |
| `xyz-wing` | XYZ翼 | 50 | 枢纽 {X,Y,Z} 连接两个钳子 {X,Z} 和 {Y,Z}，排除枢纽和两个钳子共同影响格中的 Z |
| `w-wing` | W翼 | 50 | 两个相同双值格 {A,B} 通过数字 A 的强链连接，排除共同影响格中的 B |

## 关键设计决策

### 1. 鱼用统一 base/cover 模型

basic-fish 使用统一的 base/cover 集合模型，而不是为 X-Wing/Swordfish/Jellyfish 分别实现。算法核心：
1. 对每个数字，构建每个 house 的候选位置列表
2. 筛选恰好有 2-4 个候选位置的 house 作为 base 候选
3. 对候选 base houses 做 N 选 N 组合，检查是否 cover 集恰好有 N 个 house
4. 如成立，排除 cover house 中非 base 交集的候选

避免了对鱼名称的硬编码，可自然扩展到 size 5+（虽然 9x9 数独不会出现）。

### 2. Skyscraper 的保守实现

single-digit-patterns 目前只实现了 Skyscraper，跳过了 2-String Kite 和 Empty Rectangle。原因是：
- Skyscraper 的链推理最直观（两个行/列共享一个列/行），false positive 风险最低
- 2-String Kite 和 Empty Rectangle 在调试中发现对部分 hard 谜题产生 bad-elimination，需更严格的 conjugacy 验证
- 计划 M3 在强/弱链图框架下统一实现这些单数字模式作为 X-Chain 的特例

Skyscraper 增加的安全检查：
- 跳过两个行/列完全相同的情况（那其实是 X-Wing，由 basic-fish 处理）
- 跳过两个端点在同一个 row/col/box 中的情况（消除误判过宽的问题）
- 通过 `PEERS_OF` 交集精确定位共同影响格

### 3. XY-wing 的 pincer 分离

枢纽 {X,Y} 的两个钳子严格按类型分离：{X,Z} 放入 xzPincers，{Y,Z} 放入 yzPincers。pincer 类型匹配后，只从两者共同影响格中排除 Z。

### 4. W-Wing 的双方向强链

支持两种连接方式：通过 digit A 或 digit B 的强链连接两个相同的双值格 {A,B}。强链要求为 conjugate pair（一个 house 中该数字恰好出现 2 次）。

## 解出率统计

数据来源：`data/ground-truth/` 各档 100 题，使用 `scripts/solve-rate.ts` 统计。

| 难度 | 解出数 | 解出率 | 健全数 |
|---|---|---|---|
| easy | 100/100 | 100.0% | 100 |
| medium | 100/100 | 100.0% | 100 |
| hard | 57/100 | 57.0% | 100 |
| diabolical | 1/100 | 1.0% | 100 |

- easy/medium 完全解出(expected)
- hard 解出 57%（余下需要 M3 的 AIC/ALS/链技术）
- diabolical 仅解出 1%（基本需要 full T3+T4 组合）
- **所有 trace 零 soundness violation**（AC-3 满足）

## 遇到的难点与处理

### 1. Skyscraper false positive 导致 soundness 失败

**现象**：hard 档 58 个 puzzle 的 trace 有 soundness violation，全部由 single-digit-patterns 引起。bad-elimination 的数量从 1 到 23 不等。

**根因**：当两个行有完全相同的候选列集合（如行 3 和行 7 都在列 {2,8} 有数字 1 候选）时，skyscraper 检测到 4 组匹配条件，产生错误的端点。端点在同一个列中导致 common peers 覆盖整个列，过度消除候选。

**修复**：
1. 跳过完全相同列/行集合的情况（留给 basic-fish 处理）
2. 要求两个端点不在同一 row/col/box 中
3. 用 `PEERS_OF` 交集而不是行列范围的笛卡尔积来确定消除位置

### 2. 2-String Kite 和 Empty Rectangle 暂缓

这两个模式在孤立测试中偶发 bad-elimination。推测原因是：
- 不要求 Kite 的 conjugate pair 严格为 2 个候选（存在多个候选时链推理不严格）
- Empty Rectangle 的 L-shape 检测不够精确
- 计划 M3 统一在强弱链框架下实现

### 3. 数据类型约束

项目配置了 `noUncheckedIndexedAccess`，导致 `digitsOf` 和数组成员访问返回 `T | undefined`。处理方式：在已知长度条件下使用 `!` 非空断言，而不是到处做防御性检查。

### 4. XYZ-Wing 的实现复杂度

XYZ-Wing 的 elimination 条件比 XY-Wing 更严格：需要被排除的格子同时被枢纽和两个钳子看到。实现时使用了 `commonPeers(pincer1, pincer2)` 再额外筛选同时被 pivot 看到的格子。
