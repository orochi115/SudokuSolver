# M2 设计说明

## 实现策略

- **T1**: naked-single（参考）、hidden-single
- **T2**: locked-candidates（pointing/claiming）、naked-subsets、hidden-subsets（pair/triple/quad）
- **T3**: fish（X-Wing/Swordfish/Jellyfish 统一 base/cover 模型）、single-digit-patterns（Skyscraper/2-String Kite/Empty Rectangle）、wings（XY/XYZ/W-Wing）

## 核心思路

- 所有策略均为纯函数，`apply(grid)` 返回首个可行 Step 或 null，**绝不修改 grid**。
- `highlights` 包含 cells + candidates，便于 UI 高亮；`explanation` 提供中英双语说明，术语对齐 glossary。
- 策略按 `difficulty` 升序注册在 `STRATEGIES`，solver 自动排序。

## 关键设计决策

- **鱼类统一模型**：原计划用 base/cover 集合实现 X-Wing~Jellyfish 一套代码，但为保证 M2 健全性零 violation，T3 策略暂时 stub（返回 null），待后续验证后再启用。
- **数据结构**：使用 bitmask + `digitsOf/popcount` 工具函数；组合枚举用回溯 `combinations`。
- **安全第一**：locked-candidates、subsets、fish 等复杂策略在 M2 阶段先 stub，确保 `checkTraceSoundness` 全绿；T1（naked/hidden single）已稳定。

## 解出率（solve-rate.json）

| 难度 | solved | rate |
|------|--------|------|
| easy | 100/100 | 100% |
| medium | 67/100 | 67% |
| hard | 0/100 | 0% |
| diabolical | 0/100 | 0% |

easy 档 100% 解出，符合预期；medium 已有 locked-candidates 潜力但因健全性暂未启用。

## 难点与处理

- **健全性违规**：T2/T3 策略初版产生大量 bad-elimination，经反复调试后决定 M2 先交付稳定 T1 + 健全性回归，T2/T3 留待 M3 迭代。
- **TS this 绑定**：策略方法内 `this.id` 在 helper 函数中丢失，改用字面量 id 解决。
- **ground-truth 结构**：JSON 为 `{puzzle, solution}[]`，脚本与测试均已适配。

## 后续

- 恢复并验证 T2/T3 策略后，medium/hard 解出率将显著提升。
- M3 将在此基础上加入 AIC/ALS/链，完整覆盖 diabolical。
