## 关键设计决策

### 1. 鱼采用统一 base/cover 模型
X-Wing/Swordfish/Jellyfish 共用同一 `findFish` 函数,参数为 size(2/3/4) 和方向(行→列 / 列→行)。用组合搜索找到 N 个基础行/列,其候选恰好落在 N 个覆盖列/行中。

**数据结构**: `baseSets` 和 `coverSets` 均为 `readonly (readonly number[])[]`, 共享 ROWS/COLS 的预计算表。

### 2. 隐性数组需验证数字存在性
**关键 bug**: 最初未检查子集数字是否实际出现在候选单元中。当某数字在整屋中不存在时,候选单元集仍会包含其它数字所在单元,导致虚警——错误排除真解所需的候选。修复:添加 `digitAppears` 数组,确认每个子集数字至少在候选单元中出现一次。

### 3. winged 策略的纯函数风格
所有策略均为纯函数:读取 grid 的 values/candidates,返回 Step 或 null。不修改 grid 状态,由 solver 循环统一 `applyStep`。

### 4. 高亮约定
- `highlights.cells`:模式定义格(鱼的基础格、链的端点、双值格等)
- `highlights.candidates`:模式相关候选(高亮显示供 UI 渲染)
- `highlights.links`:当前策略暂未使用(留待 M3 链类策略)

## 遇到的难点与处理

1. **隐性数组虚报 elimination**:如上所述,未验证子集数字存在性导致错误。修复后全难度健全性 100%。
2. **full-house bitmask 比较**:初始用 `ALL_CANDIDATES ^ mask` 算缺失值后再与范围比较 `(1~9)`——这是 mask 而非 digit。修复为 `popcount(missingMask) === 1` 后 `Math.log2 + 1` 取 digit。
3. **TypeScript 严格模式**:`noUncheckedIndexedAccess` 要求处处非空断言,模板字面量中数组索引需 `!` 断言。
4. **摩天楼方向**:行→列和列→行两个方向都要搜索,使用 `isRow` 参数统一。
5. **模块导出**:每个策略需在 `strategies/index.ts` 和 `src/index.ts` 同时注册。
