# M2 Design Notes — Basic Strategies Implementation

## Strategies Implemented

### T1: Singles (difficulty 10)

**full-house**: Scans all 27 houses for any house with exactly one empty cell. Computes the missing digit from a bitmask of placed digits. Returns a placement step.

**naked-single** (baseline): Already implemented as the reference strategy. Scans cells for single-candidate cells.

**hidden-single**: For each house and each digit, counts candidate positions. If exactly one cell in the house can hold that digit, places it there.

### T2: Intersections & Subsets (difficulty 20-30)

**locked-candidates**: Two sub-patterns in one strategy:
- *Pointing*: Scans each box for a digit whose candidates all lie in one row/column. Eliminates that digit from the rest of that row/column outside the box.
- *Claiming*: Scans each row/column for a digit whose candidates all lie in one box. Eliminates that digit from the rest of that box outside the line.

**naked-subset**: For each house and each subset size (2, 3, 4), finds N empty cells whose candidate union is exactly N digits. Eliminates those N digits from other cells in the house.

**hidden-subset**: For each house and each subset size (2, 3, 4), finds N digits that appear as candidates in exactly N cells. Eliminates all non-subset candidates from those N cells. **Key fix**: Only considers digits that are still candidates in the house (not already placed), avoiding a soundness bug where placed digits were incorrectly treated as "hidden".

### T3: Fish / Single-Digit Patterns / Wings (difficulty 40-50)

**basic-fish**: Unified base/cover model. For each digit and each size (2-4), tries both row-as-base and column-as-base. Finds N base houses whose digit candidates fall in exactly N cover houses. Eliminates the digit from cover-house cells outside the base intersections.

**single-digit-patterns**: Two sub-implementations:
- *Turbot family* (Skyscraper, 2-String Kite, general Turbot): Finds any two conjugate pairs (in rows/columns) connected by a weak link (one endpoint sees another). The "other" endpoints cannot both be false, so cells seeing both can eliminate the digit.
- *Empty Rectangle*: For each box where a digit occupies exactly 2 rows and 2 columns (with a 3rd empty row and 3rd empty column), finds a line conjugate pair with one endpoint in the box. Connects the outside endpoint through the box pattern to a "diagonal corner" cell, eliminating from cells seeing both.

**xy-wing**: Finds a bivalue pivot {X,Y} that sees two bivalue pincers {X,Z} and {Y,Z}. Eliminates Z from cells seeing both pincers.

**xyz-wing**: Finds a trivalue pivot {X,Y,Z} that sees two bivalue pincers sharing Z. Eliminates Z from cells seeing the pivot and both pincers.

**w-wing**: Finds two bivalue cells with the same pair {X,Y} bridged by a conjugate pair on Y. Eliminates X from cells seeing both bivalue cells.

## Key Design Decisions

1. **Fish base/cover model**: Unified implementation for X-Wing, Swordfish, and Jellyfish. A single `findFish` function handles both row-base and column-base configurations. Size is a parameter (2, 3, 4). This makes the code compact and avoids duplicating logic per fish size.

2. **Turbot family as unified pattern**: Skyscraper, 2-String Kite, and Turbot Fish are all instances of "two conjugate pairs connected by a weak link". Implemented as a single scan over all line-pairs, with pattern classification based on house types (row+row=Skyscraper, row+col=Kite, otherwise Turbot).

3. **Empty Rectangle as separate sub-pattern**: The ER pattern uses a "virtual strong link" within the box (digit confined to 2 rows and 2 columns), which isn't a conjugate pair. Implemented separately from the general Turbot scan.

4. **Hidden-subset digit filtering**: Only digits that are candidates in the house (not already placed) are considered for hidden subsets. This was a critical soundness fix — including placed digits caused ~3000 bad eliminations.

5. **Strategy ordering**: Sorted by difficulty ascending: 10 (singles) → 20 (locked candidates) → 30 (subsets) → 40 (fish, single-digit patterns) → 50 (wings). The solver sorts defensively, but this order is the intended "systematised flow".

6. **No mutation**: Every strategy's `apply` inspects the grid and returns a Step or null, never modifying the grid. The solver loop applies steps via `applyStep`.

## Challenges & Fixes

1. **Hidden-subset soundness bug**: The initial implementation included digits already placed in the house as part of "hidden subset" digit combinations. This caused eliminations of candidates that were actually the solution digit. Fixed by filtering `availableDigits` to exclude digits whose bitmask is already set in the house's placed values.

2. **Fish isInBase bug**: The initial fish implementation checked `COL_OF[c]` for row-based fish and `ROW_OF[c]` for column-based fish when determining whether a cell was in the base houses. This was inverted — should check `ROW_OF[c]` for row-base and `COL_OF[c]` for column-base. This caused the fish to eliminate candidates from cells that should have retained them.

3. **2-String Kite over-constraint**: The initial implementation required a conjugate pair in the box, which is unnecessary. The 2-String Kite only needs one cell from each line-pair to be in the same box (seeing each other). Simplified by using the general Turbot pattern.

4. **Empty Rectangle complexity**: Multiple incorrect implementations were tried before arriving at the correct one. The key insight is that the ER pattern combines a conjugate pair in a line with a "virtual strong link" in the box (digit confined to 2 rows × 2 columns). The elimination connects the outside endpoint of the line pair through the box pattern to a diagonal corner cell.

5. **TypeScript strict mode**: `noUncheckedIndexedAccess` required careful handling of array element access (using `!` assertions after length checks) and tuple types for destructuring patterns.
