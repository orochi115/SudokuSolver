# M2 设计说明

## 已实现策略及核心思路

### T1 策略
- **full-house**: 当一个单元（行/列/宫）中只剩一个空格时，填入该单元缺少的唯一数字。
- **hidden-single**: 当某个数字在一个单元中只有一个可能位置时，填入该数字。

### T2 策略  
- **locked-candidates**: 区块排除策略，包括指向（pointing）和声明（claiming）两种形式。
- **naked-subset**: 显性数组，包括数对、三数组、四数组（统一实现）。
- **hidden-subset**: 隐性数组，包括数对、三数组、四数组（统一实现）。

### T3 策略
- **basic-fish**: 基础鱼，使用统一的 base/cover 模型实现 X-Wing、剑鱼、水母。
- **single-digit-patterns**: 单数字模式，包括摩天楼、双线风筝。
- **xy-wing**: XY翼，利用三格形成删除。
- **xyz-wing**: XYZ翼，XY翼的扩展版本。
- **w-wing**: W翼，两个相同双值格通过强链连接。

## 关键设计决策与取舍

1. **鱼的统一模型**: 使用 base/cover 集合模型统一实现 X-Wing/Swordfish/Jellyfish，便于扩展和维护。

2. **数组策略统一实现**: naked-subset 和 hidden-subset 使用参数化方法处理不同大小（2-4），减少重复代码。

3. **性能优化**: 在搜索策略时，优先检查更容易找到的模式，如先检查较小的数组和鱼。

## 遇到的难点与处理方式

1. **XY-Wing 逻辑复杂性**: XY-Wing的实现需要注意三点之间的连接关系和共同影响的格子，确保逻辑正确性至关重要。

2. **性能与完整性平衡**: 对于某些复杂的隐藏模式，采用了启发式搜索，优先检查更常见的模式，避免过度计算。

3. **边界情况处理**: 特别注意了空单元格、零候选等情况的处理，确保策略的健壮性。

4. **验证与调试**: 通过大量测试验证策略的正确性，确保不会出现违反数独规则的推理。
