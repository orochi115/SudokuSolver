# M2 Design Notes

## Implemented Strategies

### T1 (difficulty 10)
- **naked-single** (reference, pre-existing): cell with one candidate → place it.
- **full-house**: house with 8 solved cells → last empty cell = missing digit.
- **hidden-single**: digit has exactly one possible cell in a house → place it.

### T2 (difficulty 20–30)
- **locked-candidates** (区块排除, difficulty 20): pointing + claiming.
  - Pointing: all digit candidates in a box confined to one row (or col) → eliminate from rest of that row/col.
  - Claiming: all digit candidates in a row/col confined to one box → eliminate from rest of that box.
- **naked-subset** (显性数组, difficulty 30): N cells in a house contain exactly N digits → eliminate those N digits from other cells in the house.
- **hidden-subset** (隐性数组, difficulty 30): N digits appear only in N cells in a house → eliminate other digits from those cells.

### T3 (difficulty 40–50)
- **basic-fish** (基础鱼, difficulty 40): unified base/cover model for X-Wing/Swordfish/Jellyfish. For a digit, N base houses (rows or cols) have their candidates fully covered by exactly N cover houses (cols or rows) → eliminate from cover-house cells outside base.
- **single-digit-patterns** (单数字模式, difficulty 45): Skyscraper + Empty Rectangle. Currently **disabled** (stub returning null) because the elimination logic produces unsound eliminations — the idea is correct but the implementation needs more careful validation of which cells can see both endpoints of the chain.
- **xy-wing** (XY翼, difficulty 45): pivot {X,Y} sees pincers {X,Z} and {Y,Z} → eliminate Z from cells seeing both pincers.
- **xyz-wing** (XYZ翼, difficulty 45): pivot {X,Y,Z} sees pincers {X,Z} and {Y,Z} → eliminate Z from cells seeing pivot and both pincers.
- **w-wing** (W翼, difficulty 45): two bivalue cells with same pair bridged by a strong link on one digit → eliminate the other digit from cells seeing both bivalue cells.

## Key Design Decisions

### Fish — Unified Base/Cover Model
Rather than implementing X-Wing, Swordfish, Jellyfish as separate hardcoded patterns, all three share one generic base/cover set implementation. For each digit, candidate rows are collected (with 1 to size+1 candidates each to keep the fish degenerate-incomplete cases out). Combinations of `size` base rows are tried; if the union of their candidate columns equals exactly `size`, a fish pattern is confirmed and eliminations are computed. The same logic inverts for column-base fish.

### Hidden Subset — Two Invariant Checks
Hidden subsets require two simultaneous invariants: (1) the N cells' union mask has exactly N bits, and (2) each of those N digits is **restricted** to those N cells (no other house cell has any of those digits as a candidate). Only when both hold are non-subset digits eliminated from the subset cells.

### Locked Candidates — Correct "All in One House" Check
Early versions of locked-candidates checked `rowsWithDigit.size === 1` but this triggered on single candidates (which should be hidden-single, not locked-candidates). The key is: if ALL candidates for a digit in a box happen to be in one row, AND there are 2+ of them, that's pointing. If only one candidate exists, it's already a hidden-single and should not trigger pointing. The current implementation requires `rowCells.length >= 2` for pointing, and `boxesWithDigit.size === 1` with `boxCells.length >= 2` for claiming.

### Single Digit Patterns — Disabled
The Skyscraper (two conjugate pairs in two rows sharing two columns, eliminate from common column) and Empty Rectangle (digit appears twice in a box at diagonal corners, eliminate from cells seeing both) patterns are conceptually sound but the elimination logic in the implementation was not conservative enough — it eliminated candidates that the solution needed.

The root issue: a Skyscraper eliminates from cells in the *common column outside the base rows*, but only if those cells can be proven (via chain logic) to see both endpoints. The current code was over-eliminating. Rather than ship broken logic, the strategy is stubbed to return null. Re-enabling requires adding proper strong/weak link chain validation before each elimination.

## Difficulties Encountered

### Unsound Eliminations in single-digit-patterns
The most significant issue was single-digit-patterns producing eliminations that contradicted the solution. The debugging process revealed that the elimination logic for both Skyscraper and Empty Rectangle was not properly verifying that a cell truly "sees" both endpoints of the chain. A conservative fix was applied (disable the strategy entirely) pending a proper chain-validation review.

### TypeScript Non-Null Assertion Bank
The codebase uses non-null assertions (`!`) in many array indexing operations (e.g., `const r1 = rowsWithDigit[0]!`). These are safe in context but TypeScript's strict mode still flags them. The current code passes typecheck but required careful placement of assertions to avoid "possibly undefined" errors.

### Locked Candidates Requiring 2+ Candidates
The pointing pattern requires ALL candidates for a digit in a box to be in the same row. If there's only one candidate, it should be handled by hidden-single (or the solver should just place it). The `rowCells.length >= 2` guard is essential to prevent locked-candidates from stealing eliminations that should go to hidden-single.

## Unit Tests
Each strategy has a unit test in `strategies.test.ts`:
- full-house: tests placement in a row with 8 solved cells
- hidden-single: tests digit with one possible cell in a row
- locked-candidates: tests that it runs without error on an empty board
- naked-subset, hidden-subset, basic-fish, xy-wing, xyz-wing, w-wing: tests null return on empty board

A soundness regression test in `solver.test.ts` verifies zero violations across all 400 ground-truth puzzles for the full strategy set.
