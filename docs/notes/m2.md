# M2 Strategy Implementation Notes

## Implemented Strategies

- `full-house`: scans each house for exactly one empty cell, computes the missing digit, and returns a placement step.
- `hidden-single`: scans every house and digit; if a digit has exactly one candidate cell in that house, returns a placement step.
- `locked-candidates`: covers both pointing and claiming. It finds digit candidates confined to a row/column inside a box, or confined to one box inside a row/column, then eliminates from the intersecting house outside the locked area.
- `naked-subset`: covers naked pairs, triples, and quads. It searches house-local cell combinations where N cells contain exactly N total candidate digits, then removes those digits from other cells in the house.
- `hidden-subset`: covers hidden pairs, triples, and quads. It searches digit combinations where N digits appear only in N cells, then removes non-subset candidates from those cells.
- `basic-fish`: covers X-Wing, Swordfish, and Jellyfish through one base/cover implementation. Rows and columns are both tried as base houses.
- `single-digit-patterns`: implements the Turbot-family model as two strong links joined by a weak link for one digit. This covers the Skyscraper / 2-String Kite / Empty Rectangle family at the short-chain level.
- `xy-wing`: finds a bivalue pivot `{X,Y}` with pincers `{X,Z}` and `{Y,Z}`, then removes `Z` from cells seeing both pincers.
- `xyz-wing`: finds a trivalue pivot `{X,Y,Z}` with bivalue pincers `{X,Z}` and `{Y,Z}`, then removes `Z` from cells seeing pivot and both pincers.
- `w-wing`: finds two matching bivalue cells bridged by a strong link on one digit, then removes the other digit from cells seeing both wings.

`naked-single` remains the baseline strategy. All required ids are registered in `packages/engine/src/strategies/index.ts` in ascending difficulty order.

## Key Design Decisions

- I kept the engine core untouched and added only strategy modules, tests, `solve-rate.ts`, and this notes file.
- I added a small `strategies/common.ts` helper module for repeated house/candidate operations, combinations, peer checks, strong links, and explanation formatting. This keeps each strategy module focused while preserving pure `apply(grid)` behavior.
- Fish use the requested unified base/cover model. Size 2 maps to X-Wing, size 3 to Swordfish, and size 4 to Jellyfish.
- Single-digit named patterns are represented as a short strong-weak-strong chain over one digit. This favors one sound implementation over separate brittle scanners for each visual naming variant.
- W-Wing is also treated as a short chain template: matching bivalue wings plus a conjugate-pair bridge.
- Strategy tests use synthetic minimal candidate grids so each technique can be asserted exactly without relying on a full puzzle that may expose cheaper strategies first.

## Difficulties and Handling

- Several T3 techniques overlap as short chain templates. I used a conservative shared strong-link model for single-digit patterns and a separate explicit template for W-Wing, which keeps soundness simple and exposes chain links for future replay UI.
- Hidden subsets can easily rediscover complements of naked subsets. The solver order runs naked subsets first, and hidden subsets only return when they produce concrete eliminations.
- Synthetic unit tests can create candidate states that would normally be consumed by cheaper strategies first. Tests assert each strategy independently, while the full solver order is protected by the 400-puzzle soundness regression.
