# M2 Strategy Implementation Notes

## Implemented Strategies

- T1 singles: `hidden-single` complements the existing `naked-single` by scanning each row, column, and box for a digit that has exactly one possible cell.
- T2 locked candidates: `locked-candidates-pointing` handles box-to-line eliminations; `locked-candidates-claiming` handles line-to-box eliminations.
- T2 subsets: `naked-pair`, `naked-triple`, `naked-quad`, `hidden-pair`, `hidden-triple`, and `hidden-quad` share subset scanners while exporting separate `Strategy` objects.
- T3 fish: `x-wing`, `swordfish`, and `jellyfish` use one base/cover implementation for size 2, 3, and 4 fish in row and column orientations.
- T3 single-digit patterns: `skyscraper`, `two-string-kite`, and `empty-rectangle` use conjugate-line checks and only eliminate candidates outside the defining pattern.
- T3 wings: `xy-wing`, `xyz-wing`, and `w-wing` scan bivalue/trivalue patterns and eliminate only from cells that see the required wing endpoints.

Each strategy module returns the first applicable `Step`, leaves `Grid` mutation to the solver loop, and fills `highlights` plus bilingual explanations.

## Design Decisions And Tradeoffs

- Fish use a unified base/cover model. Rows and columns are both treated as base sets, and the shared cover set size determines X-Wing, Swordfish, or Jellyfish. This keeps M2 small and leaves room for M3 complex fish variants.
- Subsets use shared factories but keep one exported module per named strategy. This preserves the strategy registry as the teachable flow while avoiding duplicate combination code.
- T3 chain-like named patterns are intentionally conservative. During corpus soundness testing, broad matches could delete connector candidates. The implementation now excludes all pattern cells from eliminations and requires distinct connector cells where needed.
- Strategy order is cheap-first: singles, locked candidates, subsets, fish, then wings/single-digit patterns. This makes the trace prefer simpler human explanations when several deductions are available.
- No foundation modules were changed. The work adds strategy modules, tests, a reporting script, and generated notes/report files.

## Difficulties Encountered

- The main risk was soundness in T3 patterns. Some intuitive endpoint-based eliminations are too broad if connector cells are not excluded. The corpus regression caught this, and the strategy matchers were tightened rather than relying on downstream correction.
- Unit tests use direct candidate-mask fixtures because many strategies are candidate-elimination patterns, not placement patterns. This keeps tests minimal and deterministic without needing large puzzle fixtures.
