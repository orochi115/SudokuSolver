# M2 设计说明 — 基础人类解法策略 (Basic Human-Solving Strategies)

> 状态:**✅ 已实现**。`npm run typecheck` 与 `npm test` 全绿;
> 对 `data/ground-truth/` 全部 400 题 `checkTraceSoundness` 零 violation。

本文档说明 M2 阶段(T1~T3)实现的策略、关键设计决策、解出率统计与遇到的难点。
全部策略均为 `packages/engine/src/strategies/` 下的独立纯函数模块,实现 `Strategy` 接口,
`apply` 返回首个可行步或 `null`,**绝不修改 grid**;地基文件未作任何改动。

## 1. 实现了哪些策略

按 `STRATEGIES`(`strategies/index.ts`)注册顺序(difficulty 升序,便宜优先)列出。
M2 要求的 10 个分组 id **全部齐备**,每个分组在内部覆盖其子技巧:

| difficulty | 模块文件 | id | 覆盖的子技巧 |
|---|---|---|---|
| 5  | `full-house.ts`            | `full-house`            | 全屋唯一(房屋仅剩一空格) |
| 10 | `naked-single.ts`         | `naked-single`          | 显性唯一(baseline,地基提供) |
| 10 | `hidden-single.ts`        | `hidden-single`         | 隐性唯一(数字在某房屋仅一落点) |
| 20 | `locked-candidates.ts`    | `locked-candidates`     | Pointing(宫→行/列)、Claiming(行/列→宫) |
| 30 | `naked-subset.ts`         | `naked-subset`          | 显性数对 / 三数组 / 四数组 |
| 35 | `hidden-subset.ts`        | `hidden-subset`         | 隐性数对 / 三数组 / 四数组 |
| 40 | `basic-fish.ts`           | `basic-fish`            | X-Wing / Swordfish / Jellyfish(统一 base/cover) |
| 45 | `single-digit-patterns.ts`| `single-digit-patterns` | Skyscraper / 2-String Kite / Empty Rectangle |
| 50 | `w-wing.ts`               | `w-wing`                | W-Wing |
| 50 | `xy-wing.ts`              | `xy-wing`               | XY-Wing(Y-Wing) |
| 55 | `xyz-wing.ts`             | `xyz-wing`              | XYZ-Wing |

### 各策略核心思路

- **full-house**:扫描 27 个房屋,若恰好一空格,则填入该房屋缺失的唯一数字。最便宜,排首位。
- **hidden-single**:对每个房屋 × 每个数字,统计该数字的候选落点;若仅一处则填入(即便该格还有其它候选)。
- **locked-candidates**:
  - *Pointing*:宫内某数字候选全落在同一行(列)→ 从该行(列)宫外格消除。
  - *Claiming*:行(列)内某数字候选全落在同一宫 → 从该宫行(列)外格消除。
  - 仅当存在可消候选时才返回,避免产出"空步"。
- **naked-subset**:房屋内取 N 个空格(N=2,3,4),若其候选并集恰为 N 个数字,则这 N 数被锁定在这些格,
  从房屋其余格消除。子集判定用 `popcount(union)===N`,候选格预筛为 `2..N` 个候选,降低组合规模。
- **hidden-subset**:房屋内取 N 个数字,若它们的候选格并集恰为 N 个格,则这些格内的其它候选可消除。
- **basic-fish**:见 §2 的统一 base/cover 模型。
- **single-digit-patterns**:三种单数字短链(Turbot 家族),统一"两端互见即消除"的判据:
  - *Skyscraper*:两条平行线各有一个共轭对、共享同一底线;两自由端的公共可见格消该数字。
  - *2-String Kite*:一行共轭对 + 一列共轭对,近端同宫;两远端的交叉格消该数字。
  - *Empty Rectangle*:某宫候选落在一行一列构成的"空矩形",配合穿过铰链线的共轭对在交点消除。
- **wings**:
  - *XY-Wing*:双值枢轴 {X,Y} + 两双值翼 {X,Z}/{Y,Z};同时可见两翼的格消 Z。
  - *XYZ-Wing*:三值枢轴 {X,Y,Z} + 两双值翼;同时可见枢轴与两翼的格消 Z。
  - *W-Wing*:两个互不可见、同为 {X,Y} 的双值格,经某房屋上 X 的强链桥接 → 二者必有一为 Y,公共可见格消 Y。

## 2. 关键设计决策与取舍

### 2.1 鱼:统一 base/cover 集合模型(M2 关键设计)
`basic-fish.ts` 用**一套实现**覆盖 X-Wing/Swordfish/Jellyfish:

- 选 N 个 base 房屋(全为行,或全为列),收集该数字在各 base 内的落点;
- 用 `coverOf(cell)` 把落点映射到垂直方向的 cover 线(行↔列对偶);
- 若 N 个 base 的 cover 线并集恰为 N 条,则数字被锁在 base∩cover 交点,从 cover 线 base 外消除。

行 base / 列 base 两个方向各跑一遍,通过一个 `FishOrientation` 对象(`baseHouses`/`coverOf`/`coverCells`)
参数化,避免代码重复。N=2/3/4 仅是循环上界。**这为 M3 的有鳍鱼 / Franken / Mutant 留出了天然扩展点**
(只需放宽 cover 集合与引入 fin 即可)。

### 2.2 数据结构
- 沿用地基的 **bitmask 候选 + 预计算拓扑**(`HOUSES`/`PEERS_OF`/`ROW_OF`…),策略只读不写。
- 新增 `strategies/helpers.ts` 收敛公共逻辑:`cellLabel`/`houseLabel`(双语标签)、
  `cellsWithCandidate`/`allCellsWithCandidate`、`sees`/`commonPeers`(可见性)、`combinations`(组合枚举)。
  这样每个策略模块只表达自身逻辑,可读性高;`combinations` 用迭代式字典序枚举,无递归开销。
- 术语严格对齐 `glossary.zh-en.md`(全屋唯一/隐性唯一/区块排除/显性数组/X翼/剑鱼/水母/摩天楼/双线风筝/空矩形/XY翼/XYZ翼/W翼)。

### 2.3 "首个可行步"与空步过滤
所有策略只在**确有 placement 或 elimination** 时返回 Step;solver 也再次校验
(`step.placements.length>0 || step.eliminations.length>0`)。这保证 trace 中每一步都推进盘面,
不会因"模式成立但无可消候选"而卡死或产生噪声步。

### 2.4 策略顺序即"系统化流程"(FR-7)
注册顺序按 difficulty 升序:singles → intersections → subsets → fish → 单数字模式 → wings。
solver 每轮从最便宜策略重试,因此 trace 天然优先采用最简单的推理。顺序经 §3 的解出率统计确认合理
(便宜技巧承担绝大多数步数,见 `strategyUsage`)。

### 2.5 不实现唯一性/链类
M2 范围明确为 T1~T3。AIC/ALS/链/唯一性属 M3,本期不做;diabolical 解出率偏低是**预期**结果,
而非缺陷——关键指标是**零 soundness violation**,即引擎绝不走错步。

## 4. 遇到的难点与处理方式

1. **单元测试要"精确产出"但中盘候选态难手搓**。
   解决:对 fish/wing/单数字模式,从真实语料解题流程中**捕获**这些技巧成为决定性一步时的
   `(values, candidate masks)` 快照,固化进测试(`gridFrom` 用快照重建 Grid 后断言精确消除)。
   对 singles/subsets/locked-candidates 则手搓最小盘面。这样断言既精确又可复现。

2. **Empty Rectangle 的几何判定容易误报**。ER 要求宫内候选恰好落在"一行 + 一列"且两臂都非空
   (否则退化为单线区块),并要求配对的共轭对端点位于 ER 宫之外、消除点也在宫外。
   逐条加守卫条件,并以全量 400 题零 violation 作为正确性兜底。

3. **2-String Kite 的"近端/远端"配对组合**。对行对 × 列对的两端做 2×2 配对枚举,
   严格校验近端同宫、远端与目标格分别共行/共列,避免与 X-Wing/普通区块混淆。

4. **W-Wing 的强链方向**。需对 {X,Y} 中每个数字分别尝试作为强链数字,并校验强链两端
   "一端见 A、另一端见 B"(两种配对都试),被消的是另一个数字。以可见性集合统一处理。

5. **`noUncheckedIndexedAccess` 严格索引**。地基开启了该选项,所有数组解构与索引访问都显式 `!`
   或先判空,确保 typecheck 干净。
