# M2 设计说明

## 本次实现的策略

已按要求新增并注册以下策略 id（`packages/engine/src/strategies/index.ts`）：

- `full-house`
- `hidden-single`
- `locked-candidates`（包含 Pointing / Claiming）
- `naked-subset`（Pair / Triple / Quad）
- `hidden-subset`（Pair / Triple / Quad）
- `basic-fish`（X-Wing / Swordfish / Jellyfish）
- `single-digit-patterns`（Skyscraper；2-String Kite / Empty Rectangle 在模块内保留扩展位）
- `xy-wing`
- `xyz-wing`
- `w-wing`

另外保留并继续使用已有基线策略 `naked-single`。

## 核心思路

1. 单数类（T1）
   - `full-house`：扫描每个 house，若仅一个空格，则缺失数字直接填入。
   - `hidden-single`：扫描每个 house 的每个数字，若该数字只出现在一个候选位置，则填入。

2. 区块与数组（T2）
   - `locked-candidates`：
     - Pointing：宫内某数字候选落在同一行/列，删除该行/列宫外同数字。
     - Claiming：行/列某数字候选全落在同一宫，删除该宫内该行/列外同数字。
   - `naked-subset`：在 house 中寻找 k 个格的候选并集恰为 k 个数字（k=2/3/4），据此消去 house 其他格中的这些数字。
   - `hidden-subset`：在 house 中寻找 k 个数字只落在 k 个格（k=2/3/4），据此删除这 k 个格中的其他候选。

3. 鱼/模式/翼（T3）
   - `basic-fish`：统一 base/cover 模型实现 size 2/3/4 鱼（X-Wing / Swordfish / Jellyfish），行基和列基双向搜索。
   - `single-digit-patterns`：当前以保守且通过 AC-3 的 Skyscraper 实现为主。
   - `xy-wing` / `xyz-wing` / `w-wing`：分别按 pivot+pincers、tri-value pivot、bivalue twins + conjugate bridge 建模并做交集消除。

## 关键设计决策与取舍

1. 鱼类统一建模
   - 采用统一 `findFish(grid, digit, size, orientation)`，避免 X-Wing / Swordfish / Jellyfish 三份重复代码。
   - 扩展复杂鱼时，只需增加 size 或约束层，不改主框架。

2. 子技巧聚合策略 id
   - 根据里程碑要求按固定 id 聚合子技巧（如 `naked-subset`、`basic-fish`），降低注册面和回放层耦合。

3. soundness-first
   - 对易误判模式采取保守实现，优先保证 AC-3（零 violation）。
   - `single-digit-patterns` 模块保留 2-String Kite / Empty Rectangle 的扩展函数位，后续在 M3 可继续强化并加更多判定约束。

4. 测试策略
   - 每个策略用手工最小候选盘面做单测，断言首个 Step 的 `strategyId` 与精确 `placements/eliminations`。
   - 另加 400 题回归测试，确保全策略组合在真实语料上无不健全步骤。

## 遇到的难点与处理

1. 单数字模式误判风险高
   - 早期版本在语料回归中触发大量 soundness violation。
   - 通过收紧条件并回归验证，最终保留可靠子集，确保 AC-3 全绿。

2. 高级策略组合下的首步不稳定
   - 由于 `apply` 必须返回“首个可行步”，测试盘面容易触发旁路模式。
   - 通过最小候选盘面隔离目标结构，保证测试精确且稳定。

3. 性能与可维护性平衡
   - T2/T3 有组合搜索（subset/fish），通过按 house 和 digit 分层、先做快速剪枝（候选数量约束）减少遍历开销。
