# M3 Design Notes

## Link Graph And AIC Model

The AIC engine models each candidate as a node `{ cell, digit }`. Edges are typed as `strong` or `weak`, matching `trace.Link`:

- Cell links: candidates inside a bivalue cell form a strong link; all same-cell candidate pairs are weak links.
- House links: a digit with exactly two candidate cells in a house forms a strong link; any same-digit candidates that see each other form weak links.
- Search: DFS walks alternating strong/weak links, starts and ends on strong links, and looks for same-digit endpoints. If an outside candidate sees both endpoints, it is eliminated.

Grouped links are kept internal for now because `trace.Link` is intentionally pairwise. For visualization, grouped reasoning is represented by the decisive pair links that justify the elimination.

## Added Strategies

- `simple-coloring`: builds single-digit conjugate components and handles trap/wrap. Wrap removes a color when same-color nodes see each other; trap removes outside candidates seeing both colors.
- `aic`: covers X-Chain/XY-Chain/Nice Loop style eliminations through the unified candidate graph.
- `als`: implements ALS-XZ with restricted common candidates. It enumerates small almost locked sets inside houses and removes shared candidates seen by all relevant occurrences.
- `uniqueness`: implements Unique Rectangle Type 1 and BUG+1. It is registered late and documented as relying on the unique-solution assumption.
- `sue-de-coq`: implements a conservative row/box intersection pattern that locks line-only and box-only candidates.
- `forcing-chain`: implements bounded single-premise propagation only. It is a last-resort strategy and never calls the brute-force solver or ground truth.

## Forcing Boundary

`docs/forcing-boundary.md` defines the distinction: a human-acceptable forcing chain has one premise, linear forced consequences, bounded depth, and a visible contradiction or common result. Multi-branch nets, Nishio, template enumeration, and backtracking are excluded from default solving.

The current `Strategy` interface has no runtime configuration argument. The implementation therefore exposes the boundary through conservative defaults and strategy registration. A future config-aware solver can disable `forcing-chain` or swap it for broader optional fallback strategies without changing trace consumers.

## Hardest Part

The hardest part was making chain search useful without compromising soundness. The implementation keeps the graph small and deterministic, only accepts standard same-digit endpoint eliminations, and relies on the 400-puzzle ground-truth soundness test to catch any bad placement or elimination. For forcing, the implementation deliberately stops short of forcing nets: it only removes a candidate when one assumption causes a bounded local contradiction.
