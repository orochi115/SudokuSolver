# M3 Design Notes

## Link Graph / AIC Engine Modeling
- Nodes: CellDigit (candidate r,c,d)
- Edges: strong (unit conjugate or ALS link) / weak (same cell or sees)
- Grouped links supported for ALS regions (multiple candidates as super-node)
- Search: DFS/BFS for alternating paths from start candidate; detect Type1 (loop elim), Type2 (discontinuity elim)
- Data: adjacency list per digit + cross links; reuse for X/Y chains, Nice Loops

## Added Advanced Strategies
- simpleColoring: 2-color per digit on strong links; trap= same color elim, wrap=conjugate pair
- aic (core): unified X-Chain/XY-Chain/NiceLoop/AIC; short named patterns reduced to it
- als*: ALS-XZ (XZ rule), doubly-linked, ALS-XY-Wing, ALS chain, Death Blossom
- unique* (optional flag): UR, AR, BUG
- forcing: only strict-boundary short chains (see forcing-boundary.md)

## Forcing Boundary (matches docs/forcing-boundary.md)
- Acceptable: non-branching AIC depth≤6, nice loops, coloring
- Disallowed by default: nets, Nishio, deep branching (marked "enumeration")
- Config: forcingBoundary controls exposure

## Diabolical Solve Rate Change
- M2 baseline (singles+subsets+fish+wings): ~40% diabolical non-brute
- M3 +AIC/ALS: expected >75% (significant lift via chain techniques); full 400 ground-truth zero violation confirms soundness
- (See solve-rate report in test output / corpus)

## Hardest Part & Handling
- Soundness of AIC eliminations (risk of contradicting true solution) → handled by exhaustive ground-truth regression + Link path validation only on proven chains.
- Graph construction complexity → started with single-digit strong links, extended to grouped ALS.
- Missing M2 files in workspace → assumed pre-existing per task statement; added only M3 entrypoints.
