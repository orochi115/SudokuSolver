## 一、链接图 / AIC 引擎建模方式

### 节点模型
AIC 引擎以 **候选数** 为节点：`(cell, digit)` 对。81×9 = 最多 729 个节点。  
内部用整数 `cell * 10 + digit` 编码（范围 0–809），存入 `Set<number>` 进行 O(1) 访问与去重。

### 强/弱链定义

**强链 Strong Link** （`from` 为假 → `to` 为真）：
1. 同一宫/行/列中，某数字的候选只有 2 个（共轭对）
2. 同一格中，候选数恰好为 2（双值格），另一个候选数即为强链对端

**弱链 Weak Link** （`from` 为真 → `to` 为假）：
1. 同数字，两格互相可见（共享宫/行/列）
2. 同格，不同数字（格内互斥）

所有强链都可以作为弱链使用（共轭对和双值格同时满足强弱）。

### DFS 搜索

```
start(strong) → n1(weak) → n2(strong) → ...
```

- `dfsStrong(current)`：从 `current` 出发，遍历强邻居 `next`，推入路径，然后调用 `dfsWeak(next)`
- `dfsWeak(current)`：从 `current` 出发，遍历弱邻居 `next`，推入路径，然后调用 `dfsStrong(next)`
- 当我们通过强链到达一个节点时（在 `dfsStrong` 中），检查端点推论

**端点推论（链结束于强链时）**：
- Type 1：start.digit == end.digit，不同格 → 消去公共可见格中的 `digit`（排除链节点本身）
- Type 2：start.digit != end.digit，同格 → 消去该格其余候选

**不连续回路（closing via strong back to start）**：
- 链回到起点 via 强链 → 起点必为真 → 填入起点

深度上限：14 个节点（覆盖实际人类求解中的绝大多数链）。

## 二、新增高级策略

### 2.1 Simple Coloring（简单染色，difficulty 60）
- 找单数字的强链连通分量，用 BFS 双色染色
- **颜色陷阱（Trap）**：某格同时看到两种颜色 → 消去
- **颜色矛盾（Wrap）**：同色两格互见 → 该色全假，另一色全真（填入）
- 实现：`simple-coloring.ts`

### 2.2 AIC（交替推理链，difficulty 70）
- 建模见上述 §一
- 覆盖：X-Chain（单数字链）、XY-Chain（双值格桥接）、Skyscraper/Kite 已由 T3 单数字模式覆盖
- 实现：`aic.ts`

### 2.3 ALS（几乎锁定集，difficulty 80）
- **ALS 发现**：枚举每个宫/行/列中 1–4 格的子集，筛选"N 格含 N+1 候选"的 ALS
- **ALS-XZ**：两个 ALS 通过受限公共候选 X（RCC）连接，消去公共候选 Z
  - 严格 RCC 检查：ALS_A 的所有 X-候选都看到 ALS_B 的所有 X-候选
  - Z 的消去：找到能看到 ALS_A 所有 Z AND ALS_B 所有 Z 的外部格
- **ALS-XY 翼**：三个 ALS 通过两个 RCC 链接
- **死亡之花（Death Blossom）**：严格限制茎格为双值格（恰好 2 候选），每个候选对应一个 ALS 花瓣
- 实现：`als.ts`

### 2.4 Uniqueness（唯一性，difficulty 90）
- **UR Type 1**：三角确认 UR 对，消去第四格的 UR 对
- **UR Type 2**：两底层格各有一个额外候选且相同 → 消去公共可见格
- **UR Type 4**：一个 UR 数字在底层格所在宫/行/列中被锁定 → 消去另一个 UR 数字
- **BUG+1**：全局唯一的三值格，找 odd-count 数字并填入
- 标记为 optional（依赖唯一解假设），通过 STRATEGIES 注册控制
- 实现：`uniqueness.ts`

### 2.5 Sue de Coq（苏德蔻，difficulty 95）
- 2-3 格交叉区间（行/列 ∩ 宫）
- 严格验证：行/列部分的数字在该行/列中 **仅出现** 在交叉区间 + 行伴侣格；宫部分同理
- 行部分形成裸对/三，宫部分形成裸对/三
- 关键验证（避免旧版 bug）：`L_confinedInLine` / `B_confinedInBox` 检查保证数字真正被锁定
- 实现：`sue-de-coq.ts`

### 2.6 Forcing Chain（强制链，difficulty 100）
- 仅传播**裸单**（Naked Single），禁止隐藏单数（避免隐式依赖链）
- **单格强制链**：2-4 候选格，各假设各传播，找矛盾（消去）或一致结论（填入）
- **宫/行/列强制链**：某数字恰好 2 个位置，分别假设，找矛盾或一致结论
- 传播上限：50 步，防止无限循环
- 实现：`forcing-chain.ts`

## 三、Forcing Chain 边界取舍

参见 `docs/forcing-boundary.md` 完整规则。关键取舍：

| 技术 | 取舍 | 原因 |
|------|------|------|
| 裸单传播 | ✅ 允许 | 简单、确定、可追踪 |
| 隐藏单传播 | ❌ 禁用 | 实现复杂，曾导致健全性违反 |
| 矛盾回路 → 消去 | ✅ 允许 | 结论局限于假设本身 |
| 多分叉强制网络 | ❌ 禁用 | 非线性，不可追踪 |
| Nishio | ❌ 禁用 | 本质是暴力搜索 |

强制链处于边界地带（difficulty=100），是引擎"最后手段"。

## 五、最难的部分与处理方式

### 5.1 AIC 健全性
**问题**：初版 AIC 有三个 bug：
1. Type 1 消去时没有排除链节点本身（链节点可能出现在 commonPeers 中）
2. 弱链闭合到起点 → 错误地"消去起点"（逻辑不成立）
3. 同格但不同数字的弱链节点未正确建模

**解决**：
1. 在 `checkType1` 中用 `chainCells: Set<number>` 排除路径节点
2. 移除弱链闭合回起点的推论（只保留强链闭合 → 起点为真）
3. 严格的 `getWeakNeighbors` 和 `getStrongNeighbors` 函数，确保链类型正确

### 5.2 Sue de Coq 的错误分区
**问题**：初版允许"行部分"中包含在行其他格中仍有候选的数字，导致不合法的消去。

**解决**：增加 `L_confinedInLine` 和 `B_confinedInBox` 严格检查，要求候选数字仅出现在指定集合中。

### 5.3 ALS 死亡之花的非空约束
**问题**：初版茎格有3个候选时，选2个候选配对找花瓣；对花瓣的 RCC 判断因为"d-cells为空集"而vacuously成立，导致错误的 ALS 链。

**解决**：限制死亡之花仅对**双值茎**有效，并在 RCC 判断中加 `d1Cells.length === 0 → false`。
