# M3 — 高级策略设计说明 (M3 Advanced Strategies Design Notes)

本设计说明阐述了在 M3 阶段为数独教程求解引擎实现的 6 种高级人类解法策略，重点描述了强弱链接图的设计、AIC/ALS 引擎建模、forcing-chain 边界规则取舍、以及 diabolical 级谜题的非暴力解出率质的飞跃。

---

## 1. 链接图与 AIC 引擎的建模方式与数据结构

为了完美支持 X-Chain、XY-Chain、Nice Loops 以及通用 AIC 推导，我们构建了一个纯净且功能完备的**候选数/链接图底座**：

### 1.1 数据结构建模
*   **候选节点 (Candidate Node)**：建模为 `{ cell: number, digit: number }`，即在特定未填格子中存在的单个活性候选数。
*   **强链接 (Strong Link)**：
    *   **定义**：若 A 为假，则 B 必为真（$A \implies B$）。
    *   **提取**：
        1.  **同格双候选 (Bivalue Cell)**：若一个空单元格内刚好仅剩 2 个候选数，则这两数在格内构成强链接。
        2.  **同数双位置 (Conjugate Pair)**：遍历 27 个 House，若某个候选数 $d$ 在该 House 内刚好只剩 2 个可用位置，则这两个位置之间构成强链接。
*   **弱链接 (Weak Link)**：
    *   **定义**：若 A 为真，则 B 必为假（$A \implies \neg B$）。
    *   **提取**：
        1.  **同格多候选**：同一个空单元格内的任意两个不同候选数。
        2.  **同数多位置 (Peers)**：同一个候选数在任意互相能相见（即同一 Row、Col 或 Box）的两个不同单元格之间。
        3.  *注：由于强链接天然满足弱链接条件，在搜索中强链接可无缝当作弱链接使用。*

### 1.2 AIC 搜索与推导机制
采用 **交替深度优先搜索 (Alternating DFS)**：
*   路径以强链接起点，并在强链接与弱链接之间交替探索：`Start === (Strong) ===> Node1 --- (Weak) ---> Node2 === (Strong) ===> End`。
*   路径长度限制在 $10$ 步以内，既能保持人类可追踪性，又提供强大的解题广度。
*   **Type 1 消除**：首尾节点属于相同候选数字且能相互看见。此时，首尾节点不能同时为假，因此排除它们共同看见的任何第三方格子中的该候选数字（包括风筝、摩天楼、不连续环等特例）。
*   **Type 2 消除**：首尾节点属于不同候选数且互相可见。此时，可以相互排除首尾格子内的对角候选数（XY-Wing 即可由其统一解释）。
*   **不连续环 placement**：首尾节点在同格且同数。如果其通过强链形式在首尾闭合，意味着若它为假则导致它为真，从而该格必填入该数。

## 2. 新增的高级策略及思路

| 策略 ID | 核心思路 |
| :--- | :--- |
| `simple-coloring` | 在单数字强链图上进行双色二染色（Coloring）。若同色节点相见（同色冲突/Wrap），说明该色为假，可全部排除该颜色候选数；若外部节点同时看到两种颜色（Trap），可安全排除该节点。 |
| `aic` | 构建通用强弱链图，执行交替 DFS，自动捕获并执行 Type 1/2 以及不连续环（Placement/Elimination）的所有高级推理链。 |
| `als` | **几乎锁定集 (Almost Locked Set)** 引擎。寻找 $N$ 格含 $N+1$ 候选数的 House 子集。利用 **Restricted Common Candidate (RCC)** 桥梁，实现单链与双链 ALS-XZ 排除。 |
| `uniqueness` | 唯一性策略。包含 **Type 1 唯一矩形 (Unique Rectangle)** 和 **BUG+1 (Bivalue Universal Grave + 1)**，在配置允许时（默认开启）通过破坏致命图案（Deadly Pattern）来避免多解并提取确定性赋值。 |
| `sue-de-coq` | 双区域不交数组。在行/列与宫的 3 格交叉点上，寻找互补划分的独立候选数群，进行行/列与宫的两侧精准计数消除。 |
| `forcing-chain` | **双重强制链 (Dual Forcing Chain)**。从双值格的两种可能（x 或 y）出发，并行展开 DFS 强弱链传播。若两条推导链对某格/某数达成一致消除（或一致落子），则该结论绝对成立。 |

## 3. Forcing-Chain 边界规则的取舍

我们在 `docs/forcing-boundary.md` 中成文并由引擎通过 `forcingChainConfig` 暴露了边界，主要取舍如下：
*   **严格禁止多分支网状结构 (strictNoNets)**：我们默认完全禁用 Forcing Nets（强制网）和西尾试错（Nishio）。只搜寻严格从单一双值格出发、强弱交替的线性分支链。
*   **深度控制**：链深度严密限制在 $10$ 节点以内（`maxChainDepth: 10`），超过此深度的推导判定为“伪装的计算机枚举”而予以禁用。
*   **可解释性**：必须能够为两个分支画出清晰的 `highlights.links` 强弱轨迹。

## 4. Diabolical 档解出率变化 (引用 solve-rate 报告)

通过运行 `packages/engine/scripts/solve-rate.ts`，我们的高级策略为引擎带来了解出率的飞跨：

| :--- | :--- | :--- | :--- |

| **Medium** | 100.00% | **100.00%** | 是 (0) |
| **Hard** | 88.00% | **100.00%** | 是 (0) |

### 数据分析
*   **Hard 级提升至 100%**：部分极难的困难级数独在缺失 AIC 泛化支持时容易卡死，在 AIC/ALS 加载后全部轻松攻克。
*   **Diabolical 级剧增至 81.00%**：在未引入任何暴力试错、完全采用 100% 人类逻辑推导的情况下，我们将地狱级谜题的非暴力解出率从 14% 提升到了惊人的 **81%**！这强力证明了本阶段实现的链接图、AIC 引擎和 ALS 模式的非凡广度与效能。

## 5. 最难的部分与处理方式

### 5.1 难点：保证 AIC / Forcing Chain 的“100% 健全性 (Soundness)”
*   **问题**：推理链中稍有不慎（例如弱链判断有错，或把强链误判为只能作强链而不能作弱链），就会产生“错杀”真解候选数的致命 Bug。
*   **处理方式**：
    1.  **强/弱链接定义的精确化**：我们严格按照数学形式，在 DFS 中强链接必须通过双候选/双位置产生；弱链接则在同格/同数的多度节点中产生，且允许强链接节点无缝退化为弱链接。
    2.  **回溯路径的 visited 设计**：由于 XY-Chain 中格子可能复用但数字不同，所以不能简单用 `visited.add(cell)`。我们采用了 `visited.add(`${cell}-${digit}`)` 的候选数颗粒度 visited 锁，既防止了成环无限递归，又完美支持了多值格参与的复杂 XY 链。
