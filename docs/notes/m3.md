# M3 设计说明

## 1) 链接图 / AIC 引擎建模与数据结构

核心新增文件:

- `packages/engine/src/strategies/_links.ts`
- `packages/engine/src/strategies/aic.ts`
- `packages/engine/src/strategies/forcing-chain.ts`

建模方式:

- **节点**:候选 `(cell, digit)`。
- **边**:
  - `weak`:互斥关系(同格不同数,同宫同数)。
  - `strong`:共轭关系(同宫某数仅两候选,或双值格仅两候选)。
- **图结构**:
  - `nodes: CandidateNode[]`
  - `edges: LinkEdge[]`
  - `adjacency: LinkEdge[][]`
  - `nodeIdByCellDigit: Map<string, number>`

`aic` 实现:

- 在候选图上做强/弱交替 DFS,搜索 `strong ... strong` 的同数字端点链(Type 1)。
- 端点同数字且两端可见交集中的候选做消除。
- `highlights.links` 记录链路径,用于可视化。

`forcing-chain` 实现:

- 将强/弱边转为**蕴含图**(literal graph):
  - weak: `A=true -> B=false`
  - strong: `A=false -> B=true`
- 搜索 `A=true => A=false`(消除)与 `A=false => A=true`(落子)两类结论。
- 受 `strategy-options.ts` 中的 `forcingBoundary` 控制深度与分叉许可。

## 2) 新增高级策略与思路

已新增并注册(精确 id):

- `simple-coloring`:单数字强链双色,支持 trap / wrap。
- `aic`:交替推理链(Type 1 端点消除)。
- `als`:ALS 组合(含 ALS-XZ / doubly-linked ALS 统一判定框架)。
- `uniqueness`:Unique Rectangle / Avoidable Rectangle / BUG+1(开关控制)。
- `sue-de-coq`:线-宫交叠候选拆分消除(开关控制,默认关闭)。
- `forcing-chain`:基于蕴含链的矛盾消除与定值。

另补充:

- `packages/engine/src/strategy-options.ts`:统一暴露高级策略边界/开关。

## 3) forcing-chain 边界取舍

与 `docs/forcing-boundary.md` 对齐:

- 默认仅允许**线性单前提链**(`allowBranching=false`)。
- 限制最大链长(`maxChainLength`,默认 12)。
- 明确不把 forcing nets / Nishio / 模板枚举作为默认教学逻辑。

取舍理由:

- 优先保证可解释性和可回放性,避免“看似逻辑实则枚举”。
- 通过统一配置保留后续实验空间,但默认边界偏保守。

## 5) 最难部分与处理方式

最难点是**链式策略健全性**:

- 难点 1:强/弱链条件放宽会迅速引入不健全消除。
- 难点 2:特殊技巧(尤其 Sue de Coq、唯一性)在全语料上易出现误判。

处理:

- 以 `soundness-regression.test.ts`(400 题)为硬门槛反复收紧条件。
- 将争议较大的策略做成可配置开关(唯一性/Sue de Coq 默认关闭)。
- 高级链统一输出 `links` 可视化路径,便于 trace 复核与后续教学解释。
