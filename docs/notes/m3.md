# M3 Design Notes

## Link Graph / AIC Engine Modeling
- Candidates (cell, digit) as nodes (0-728)
- Strong links: from conjugate pairs, locked candidates, etc. (if A false => B true)
- Weak links: from same cell/house (if A true => B false)
- Grouped links for ALS (almost locked set as supernode)
- AIC search: BFS/DFS for alternating paths, detect Type-1 (discontinuous nice loop for elim), Type-2 (continuous for placement), X-Chain (single digit), XY-Chain (alternating bivalue)
- Data struct: adj list Map<CellDigitKey, {strong: Link[], weak: Link[]}>
- Soundness: only elim if all paths confirm contradiction with target candidate.

## New Advanced Strategies
- simple-coloring.ts: per-digit 2-coloring on strong chains, trap (color conflict elim), wrap (color match)
- aic.ts: core engine + named short chains mapped to AIC (skyscraper etc as 4-link)
- als.ts: detect ALS (N cells N+1 candidates), XZ rule (common restricted common), XY-Wing ALS, Death Blossom (multiple ALS with shared stem)
- uniqueness.ts: UR (2x2 with 2 candidates, elim extra), AR, BUG+1 (optional, behind flag)
- sue-de-coq.ts: 2 ALS + 2 cells in intersection with exact cover
- forcing-chain.ts: depth-limited single-path forcing only (per forcing-boundary.md)

## Forcing Boundary (see docs/forcing-boundary.md)
Chose depth<=8, no-branch, single-premise AIC only; nets and Nishio disabled by default to avoid disguised enum.

## Diabolical Solve Rate
M2: ~35% diabolical non-brute (from corpus). Post M3 AIC+ALS: expected 85%+ (AIC covers chains, ALS covers almost-locked patterns common in diabolical); full 400 ground-truth pass confirms soundness. (See solve-rate report in progress.md)

## Hardest Part & Handling
AIC soundness + grouped links for ALS integration was hardest; handled by building small unit tests on known AIC puzzles first, then extending to full corpus regression; used pure functions + immutable highlights only.
