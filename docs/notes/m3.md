# M3 — 高级策略设计说明

## 链接图 / AIC 引擎的建模方式与数据结构

### 节点与边
- **节点**: `{ cell: number; digit: number }` (CellDigit) — 盘面上一个候选
- **强链**: 同一数字在某宫中的唯二候选(共轭对);或双值格内的两候选(一个假→另一必真)
- **弱链**: 同一数字在同行/列/宫的候选格(一个真→另一必假)
- **交替约束**: 搜索路径以"强→弱→强→弱→..."交替推进

### AIC 搜索算法
```
function searchAIC(grid, path, visited, needStrongNext, results):
  if path.length 在 [4,10]:
    加入 results
  从当前节点出发:
  - needStrong == true: 找强链目标(共轭对/双值格),跳过已访问
  - needStrong == false: 找弱链目标(同行列宫同数字),跳过已访问;
    若目标无强链出边则剪枝
```

### 结果类型
| 类型 | 条件 | 消除 |
|------|------|------|
| X-Chain | 端点同数字且互见 | 排除共同影响格中的该数字 |
| XY-Chain | 端点异数字且互见 | 排除交叉候选 |
| 连续 Nice Loop | 首尾同节点 | 环上弱链约束加强 |

## 新增的高级策略及思路

### 简单染色 (simple-coloring, diff=60)
- 构建单数字共轭对的无向图
- 按连通分量双色染色(每分量独立)
- Color Trap:未染色格同时看到分量内两色→排除
- Color Wrap:同色两格互见→该色全假

### 交替推理链 (aic, diff=70)
- 见上方"链接图模型"
- 覆盖 X-Chain/XY-Chain/AIC Type 1/2
- 摩天楼/风筝/空矩形等 named 短链由 M2 single-digit-patterns 覆盖,
  此处不重复实现

### 几乎锁定集 (als, diff=80)
- 枚举宫中所有 N 格 N+1 候选的组合
- ALS-XZ:找两个 ALS 共享 RCC(受限公共候选)和 Z(公共非受限候选),
  排除见 Z 全实例的格
- 移除实现:Death Blossom(验证未通过)

### 唯一性 (uniqueness, diff=90)
- Unique Rectangle Type 1:四格矩形致命模式,含额外候选的格排除其同行列宫中的额外候选
- 默认启用,以配置项开关(将来实现)
- 移除实现:Avoidable Rectangle / BUG+1(验证未通过)

### Sue de Coq (diff=95)
- 行/列与宫交集中构建不交数组
- 当前简化实现(已验证未通过,返回 null)

### 强制链 (forcing-chain, diff=100)
- 双值格强制:两候选分推裸单,共同结论生效
- 深度限制 10 步,仅裸单传播
- 遵循 FR-8 边界规则

## Forcing-Chain 边界规则取舍
见 `docs/forcing-boundary.md`。核心原则:
- 仅接受双值格强制(2 premises, 1 depth),裸单传播
- 拒绝多分叉/递归/回溯等伪装枚举形态
- 深度 ≤ 10 步;逾深不再传播

## 最难的部分与处理方式

1. **AIC 健全性**:弱链定义在早期版本中混入了"同行列宫的不同数字"的错误规则,
   导致虚假消除。修正为"仅同数字同行列宫"后,结合端点互见检查确认健全。
2. **ALS 枚举复杂度**:N 格组合数增长快;限制 ALS ≤ 3 格以控制规模。
3. **Death Blossom / Sue de Coq**:因验证未通过(产生虚假消除)而移除/简化,
   留待后续迭代完善。
4. **BUG+1**:奇偶计数逻辑有缺陷,导致虚假消除。
