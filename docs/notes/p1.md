## 关键设计决策

1. **最小化基础改动**：为支持 Avoidable Rectangle，在 `Grid` 新增 `givens: readonly number[]`。策略本身保持纯函数；`givens` 在构造时记录，后续只读。该假设记录在 `QUESTIONS.md`。
2. **E4 ALS-XY-Wing 折叠**：通用 `als-chain` 已落地。`als-xy-wing` 保持原有 `tryALSXYWing` 行为并回退到 `als-chain` 搜索，避免回归测试变更，同时满足「als-xy-wing 是 k=3 特例」的规格。
3. **性能预算**：ALS 链搜索对 ALS 尺寸和 BFS 扩展数加了硬预算（`maxSize=3`，`BUDGET=10000`），保证默认测试在合理时间内完成。
4. **UR Type 3 修正**：`unique-rectangle.ts` 的 Type 3 子集判定遗漏了 UR 本位数字，导致在 `aic-with-ur` 提前触发时产生错误删数。已重写为正确的「屋顶格 + 外格」裸子集判定。

## 验证结果

- `npm run typecheck`：通过。
- `npm test`：294 / 294 通过。
- 400 ground-truth 题库（AC-3 回归）：零 soundness violation。
- diabolical 回归用例（#36186、#38116、#77633）：全部通过。
- 727 残集（`data/failing-diabolical/puzzles.txt`）：受求解速度限制，本次未跑完全量；抽样显示 human-default 较基线 0/727 有少量提升（约 1–3%）。完整 727 量表因单题耗时较高（nice-loop / ALS 链 / finned fish 主导），需后续性能优化才能稳定跑进 144s 验证窗口。

## 性能观察

- 当前 `npm test` 约 72s，主要耗时在 AC-3 ground-truth 400 题（~70s）与 M3 题库。
- `npm run solve:list -- --profile human-default` 跑 200 题约 112s；估算全 727 约 400s，超出验证窗口。
-  hotspots（按单题 profiling）：`nice-loop`、`als-xy-wing`、`als-chain`、`finned-jellyfish`、`finned-swordfish`。其中 `nice-loop` 与 finned fish 为存量策略；`als-chain` / `aic-with-als` 已通过预算和尺寸限制压到可接受范围。

## 遗留与下一步

- 727 覆盖率：P1 策略对部分残题有效，但整体杠杆率有限；后续可考虑 P2  exotic 策略（exocet、sk-loop、msls、franken/mutant fish 等）或优化求解器性能。
- 性能：如需稳定跑进 144s 验证窗口，需对 `nice-loop` 与 finned fish 做进一步预算或缓存优化。
- 单元测试：已新增 `packages/engine/test/strategies-p1.test.ts`，覆盖 P1 策略元数据、不变性与基本发声测试。
