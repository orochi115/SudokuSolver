## 各策略实现与思路

### Coloring 族（multi-coloring、3d-medusa）

`multi-coloring`（difficulty 620）和 `3d-medusa`（difficulty 640）作为 `simple-coloring` 的扩展，在 `multi-coloring.ts` 和 `3d-medusa.ts` 中独立实现。

- Multi-Coloring：对多个 X-Wing 层着色后寻找跨层矛盾或公共 peer 消去。
- 3D Medusa：在候选数层面做全局强弱链着色，命中时可消去相反颜色候选。

### Tridagon（tridagon）

在 `tridagon.ts` 中实现，difficulty 1100（exotic 族）。

**P1 阶段修复了关键 Bug**：原 Type 2（多守护者同数消去）实现在 12 格候选集包含多种非 D 候选时会产生非法消去。根本原因：Type 2 的证明要求"全部 12 格候选 ⊆ D ∪ {g}"，即只有守护数 g 是唯一的额外候选，才能断言"至少一个守护格必须取 g"。添加了此前置检查后，所有 ground-truth 健全性违规全部消除。

### ALS-Chain / AHS（als-chain, ahs）

`als-chain.ts` 实现通用 ALS 链（difficulty 880）和 AHS（difficulty 885）。

**E4 收编**：`als-xy-wing` 在语义上是 als-chain 长度为 2 的特例，保持独立注册维持向后兼容性，但文档标注其关系。

### 翼系（wxyz-wing, remote-pairs, bent-sets, broken-wing）

在 `advanced-wings.ts` 中统一实现，各自注册为独立 strategyId：
- `remote-pairs`（505）：复用 xy-chain 引擎，远端双值对特例
- `wxyz-wing`（520）：进阶翼型
- `bent-sets`（540）：Almost Locked Pair/Triple
- `broken-wing`（560）：单数字 oddagon 亲缘

### Uniqueness 扩展（uniqueness-extended.ts, uniqueness-p1.ts）

- `uniqueness-extended.ts`：Hidden UR、UR Type 3/5/6
- `uniqueness-p1.ts`：bug-lite（912）、bug-plus-n（914）、AR Type 1–4（975–978）、Extended UR（980）、Unique Loop（985）

### AIC 扩展（aic-extended.ts）

**aic-with-ur**（difficulty 770）：可正常工作，通过 worked-examples.test.ts 所有测试。

**aic-with-als**（difficulty 760）：已注册但当前返回 null（`apply` 无操作）。原因：

ALS 桥接逻辑存在奇偶性（parity）问题。经过多次修复尝试，根本问题在于 ALS 桥接步骤跨越 2 个奇偶位（weak + strong），而在 DFS 路径中只压入了 1 个节点（exitNode），导致路径奇偶与消去规则不匹配。正确的奇偶模型需要与 `aic.ts` 中的 `legacySearchAic` 完全对齐（path[0] = FALSE 起始，奇数位 = TRUE），但每次修复后仍有 puzzle 产生非法消去。

为确保健全性红线不被违反，当前选择禁用该策略（保留 ID 注册）。后续可基于正确的 ALS 链引擎重新实现。

## ALS 收编（E4）取舍

`als-chain` 作为通用 ALS 链已落地实现（880 难度），涵盖所有 ALS 节点间的推理链。`als-xy-wing`（840）在数学上是 als-chain 长度为 2 的特例，但保持独立注册：

1. **向后兼容**：现有测试和 trace 都引用 `als-xy-wing` 这个 ID
2. **调试可见性**：独立 ID 便于追踪哪些题用了哪个变体
3. **搜索成本**：als-xy-wing 的专用搜索比通用 als-chain 更高效

## 727 增量（human-default 档）

| 阶段 | human-default 解出数 | 较上阶段 delta |
|---|---|---|
| P0（基线 33 策略） | 约 8/727 | — |
| P1（+29 strategyId） | 8/727 | 0（无显著增量）|

727 残集对 P1 策略命中率较低的原因分析：
- 这些都是"diabolical"难度残集，经 P0 策略过滤后仍剩 719 道未解
- P1 策略（Tridagon、ALS-chain 等）理论上覆盖一部分，但实际搜索深度、格局频率较低
- `aic-with-als` 被禁用后少量命中机会丢失
- Tridagon 修复后 Type 2 条件更严格，触发频率下降

last-resort 档（含 forcing-chain）未能完整采集（搜索时间过长，超过测试限制），727 全集解出数无法在本阶段记录。

## 最难部分与处理方式

### 1. aic-with-als 奇偶性 Bug

最耗时的问题。ALS 桥接在 DFS 中表示为"隐式的 weak + 显式的 strong 链接"，但路径数组只追踪显式节点，导致奇偶计数偏差。每次修复后新的 bug 出现。最终决策：注册 ID 但返回 null，保证健全性红线不被触破。

### 2. Tridagon Type 2 误消去

原实现允许 12 格中有多种非 D 候选，但 Type 2 的数学证明只对"所有格候选 ⊆ D ∪ {g}"成立。修复：在 Type 2 分支前加 `allCellsSubsetOfDg` 检查。

### 3. 测试挂起（Worker OOM）

diabolical-regressions.test.ts 中 `solves diabolical #38116 with a sound trace` 在 aic-with-als 产生非法消去后，solver 陷入无效状态循环导致 1248 秒超时。修复 aic-with-als 后，该测试降至 484ms。
