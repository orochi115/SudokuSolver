## 修复的存量缺陷

### UR Type 6（`uniqueness.ts`）

原实现的两个 X-Wing 检查均有缺陷：
1. `onlyInURCells` — 使用了「peer 传播」替代，实际未检查 X-Wing 性质
2. `xWingOnLocked` — 只检查 locked 数出现在矩形格内，**未排除矩形外同 row/col 另有该候选**
3. 导致 8 道 727 题产生大量非法消除（最多 52 处/题）

修复：彻底替换为严格的 X-Wing 验证——遍历两行（非矩形列）和两列（非矩形行），确认 locked 候选**不出现**在矩形外。

### UR Type 3（`uniqueness.ts`）

原实现的 locked set 大小计算有误：
1. `needed = extraDigits.length - 2` — 公式错误。正确应为 `needed = extraDigits.length`，因为 locked set 包含 UR {X,Y} 和 extra 数字，总 cells = 2 + k，总 digits = 2 + m，locked set 要求 2 + k = 2 + m → k = m
2. 原式将外部格需求低估 2，导致非 locked set 的模式被误认，发出非法消除
3. 同时将 `popcount(restrictedMask) === allSubset.length` 改为 `popcount(unionMask) === allSubset.length`——之前 `restrictedMask` 仅含 extra 数字（不包含 UR），与总 cells 无法对齐
4. `otherCells` 过滤放宽为允许含 UR 数字的外部格参与 locked set

影响：8 道 727 题（最多 33 处非法消除/题）修正。

## 自审

- 所有策略均为纯函数，无试错/回溯/矛盾穷举（无 `solveBruteforce`/`countSolutions`/`findGroundTruth` 调用）
- 每个 elimination 基于标准人类推理模式
- 无策略内部做多分支搜索或枚举
- 所有 human-default 策略均合乎人类技巧分类

## 难点与处理

1. **UR3 locked set 大小**：一开始照抄常见参考的「extraDigits - 2」公式，未验证 locked set 需要包含 UR 数字的对齐。经健全性失败发现后重新推导了 `k = m` 公式。
2. **UR6 X-Wing 验证**：原检查过于宽松，只确认「locked 出现在矩形格」而非「locked 仅出现在矩形格」。此差异在有限消除场景下不会暴露，但在多候选盘面会产生广泛非法消除。
3. **727 解出数下降**（21→2）：旧码非法消除"解决了"部分 727 题。修正后这些题正确回到 stuck，是正确的行为——宁少不假。
