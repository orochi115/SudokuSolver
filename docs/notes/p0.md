## 二、复用 detector 与 overlap/boundaries 改动

### Finned/Sashimi Fish（`finned-fish.ts`）

- 扩展既有 fish 族：枚举 base 轴（行/列）上 N 条线的组合，在 cover 并集上取 N 子集作为有效 cover set。
- **鳍格** = base 候选不在 cover 并集中的格；要求全部鳍格落在同一 box。
- **消除** = cover 格中「能看到全部鳍格」且普通鱼也会消的格（sashimi 为鳍不在 cover 线上的退化）。
- sashimi 与 finned 共用同一 detector，按 size 发具名 ID（无独立 sashimi strategyId）。

### Chain 族（`chain-strategies.ts` + `aic-search.ts`）

- **Nice Loop（E6）**：`searchNiceLoop` 先搜不连续环（Rule 2 落子 / Rule 3 消数），再搜连续环（弱链单位上链外消数）。`aic.ts` 移除 loop close 返回，`*-loop` kind 仅由 `nice-loop` 发出。
- **XY-Chain**：`searchXyChain` = 双值格节点、`grouped:false`、端点同数字 Type-1 消数。
- **Turbot Fish（E2）**：逐 digit 调用 `searchAic` + `isTurbotFishPattern`（恰好 4 节点、3 链、S-W-S、单格节点）。与 skyscraper/kite/ER 共享 `x-chain` owner；`overlap.ts` 将 `turbot-fish` 移入 `members`，`unified: true`。

### UR 族（`ur-engine.ts`，E3）

- 将 `unique-rectangle-type-{1,2,4}` 与 `bug-plus-one` 的矩形枚举/逐型逻辑收敛到 `ur-engine.ts`；`uniqueness.ts` 仅保留薄包装。
- 新增 Type 3/5/6、`hidden-unique-rectangle`；`overlap.unified: true`（共享 engine，按型发 ID）。

### boundaries.ts

- `nice-loop`、`xy-chain` 从 `reserved` 移除，注册为正式策略。

## 五、难点与处理

### 1. Nice Loop 连续环过度消除

初版连续环对每条弱链消去多个 unit 的链外格，在 ground-truth 上产生 700+ violation。  
**修复**：弱链只取其两端节点共有的单一 house 做 Rule 1 消数；不连续环优先；连续环需严格交替校验。

### 2. UR Type 5/6 误报

全语料曾出现 ~61 violation。  
**修复**：Type 5 要求非对角角为精确 UR 对；Type 6 要求两屋顶格为精确 `{a,b}` 且对角含额外数，X-Wing 限定在矩形行列内。

### 3. UR Type 3 子集逻辑

原实现将宫内所有含伪格数字的格并入子集，导致无可消格、永不触发。  
**修复**：屋顶伪格计为 1 逻辑格，用组合搜索最小宫外格集，以 union 掩码做裸子集判定后再消去。

### 4. Turbot 与 link graph 强链

同 box 内行共轭对（如 r1c1–r1c2）被 box 分组后无法形成强链，skyscraper 形盘面在 `searchAic` 中不触发。  
**处理**：turbot 测试与最小触发例使用跨 box 行共轭（r1c1/r1c5 + r2c1/r2c5）；命名 pattern 仍由 `skyscraper` 等独立 detector 覆盖同 box 情形。

### 5. Finned fish cover 行候选数

曾限制 `covers.length <= size` 导致 sashimi 漏检。  
**修复**：仅要求 base 线有候选且 cover 并集可取 N 子集；允许 cover 线候选多于 N。
