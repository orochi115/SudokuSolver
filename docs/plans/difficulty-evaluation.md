# 逐格推算难度评估（方向记录）

> **状态：方向占位，暂不设计。** 本文件仅固定意图与依赖，详细设计待 [人类策略全覆盖路线图](./diabolical-727.md) 推进后再展开。

## 意图

为未来的**引导模式（guided mode）**提供"该关注哪里"的推荐，引擎需要给出两类难度信号：

1. **下一步逐格难度（next-step per-cell difficulty）**：在当前局面下，**触及每个格子**的最容易的可用推导有多难（例如该格能被哪种最廉价的技巧落子/消除）。用于提示"现在最该看哪几格"。
2. **全局逐格难度（global per-cell difficulty）**：在整条解题路径中，**每个格子**的角色/贡献难度（例如它在何时、靠多强的技巧才被确定）。用于呈现整盘的难度分布与关键格。

## 关键依赖

- 需要先把主引擎 `Step` / strategy 模型里**搜索顺序（search order）与难度评分（rating）解耦**——当前 `difficulty` 一个数字同时承担两职。可借鉴 HoDoKu 移植轨道里整理的 `level + baseScore`（评分）与 `index`（搜索顺序）分离思路（见 [`research/hodoku-logic/`](../../research/hodoku-logic/) 的架构决策文档），但**实现独立**。
- 评分口径应与 [`diabolical-727.md`](./diabolical-727.md) 的技巧分级（P0–P3 / `difficulty` 按识别成本）一致，避免两套难度体系。
