在 M4 回放组件的基础上,做**交互式引导 UI**:从任意盘面出发引导用户解题。

## 先读
- `docs/milestones/M5.md`(本次任务详细需求与验收清单)
- M4 的 `packages/web` 组件(棋盘 / 高亮 / 讲解面板,复用)
- `packages/engine/src/solver.ts`(单步求解:对当前 grid 按 difficulty 升序试策略,取首个产出)

## 任务
- **任意盘面输入**:支持用户手填/部分填写,由 `Grid` 重算候选。
- **渐进式提示**:对当前盘面求"下一可行 Step",分三级披露:
  1) 技巧类别 → 2) 关注区域(高亮相关 house/cells,不给答案)→ 3) 确切一步 + 讲解。
- **落子校验**:用户落子后判定对错;错误时给出可理解的原因。
- **复用**:棋盘/高亮/讲解面板复用 M4;提示数据仍是 `Step` 结构。

## 硬性约定
- 不改引擎与 `data/`、`docs/`。
- "下一可行 Step"直接复用引擎单步逻辑,使提示遵循"系统化流程"顺序。

## 完成判据
1. 能从任意(含自定义)盘面给出三级提示,最终一步与引擎一致
2. 落子校验正确,错误有原因解释
3. 用浏览器 MCP 走查:任意盘面给提示 + 故意落错子触发纠错(AC-6)
4. 复用 M4 组件,无重复解题逻辑
完成后总结:提示分级如何实现、校验逻辑、走查结果。
