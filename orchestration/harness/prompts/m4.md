在已有求解引擎的基础上,做**网页逐步回放**。

## 先读
- `docs/milestones/M4.md`(本次任务详细需求与验收清单)
- `packages/engine/src/trace.ts`(SolveTrace / Step / Highlights / Link —— 你只消费它)
- `packages/engine/src/index.ts`(可 import 的公共 API)

## 任务
新建 `packages/web`(Vite + TS),直接 `import { solve, Grid, STRATEGIES } from '@sudoku/engine'`,**无后端**。
- 9×9 棋盘组件:显示已填数字与候选数小标记。
- 回放控制:上一步/下一步/跳转;从 `SolveTrace.initial` 累积重建每步状态。
- 高亮层:渲染当前 `Step.highlights` 的 cells / candidates / links(链画成连线,强弱不同样式)。
- 讲解面板:显示当前 `Step.explanation`,支持中/英切换。
- 谜题来源:可粘贴 81 字符串,或从 `data/ground-truth/` 选题。

## 硬性约定
- 不改引擎(`packages/engine`)与 `data/`、`docs/`。
- 这一层是"哑"的:**只消费 SolveTrace,不含任何解题逻辑**。
- 组件解耦(棋盘 / 高亮 / 讲解面板),为 M5 复用预留。

## 完成判据
1. `npm run dev` 可启动 web 应用
2. 能加载任一谜题并逐步回放,高亮(含链路径)与双语讲解正确对应每步(AC-5)
3. 用 chrome-devtools / playwright MCP 走查若干题确认渲染
完成后总结:技术选型、组件结构、用了哪些题验证。
